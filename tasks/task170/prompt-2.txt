当前 totp TOTP 双因素认证的实现存在一些问题需要修复：
1. 两个异常类定义后在验证流程中从未被抛出，验证失败时使用了返回值模式而非异常模式，导致这些异常类成为死代码。需要统一 API 风格，要么让验证失败时抛出对应的领域异常，要么从公开 API 中移除未被使用的异常定义。
2. 密钥种子数据模型中包含算法字段用于声明使用的哈希算法，但底层 HMAC 计算始终固定使用 SHA-1，导致算法字段成为一个仅有声明作用而无实际效果的装饰性字段。如果将算法设为 SHA256 或 SHA512，实际计算仍然使用 SHA-1，行为与声明不一致。需要让算法字段真正驱动底层哈希算法的选择。
3. 密钥生成方法和服务初始化方法允许的最小密钥长度低于要求的下限，允许调用方指定过短的密钥长度并成功生成密钥，违反了对密钥熵的最低要求。需要收紧校验边界以匹配规定的下限。
4. 重新运行 `poetry run pytest tests/totp/ -q`，需要让所有测试通过。
