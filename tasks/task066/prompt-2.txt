你实现的频次封顶域功能模块存在一些问题，需要修复：
1. reset_subject 方法通过逐次回滚全局计数器来移除某个主体的使用量，但回滚机制的设计意图是撤销最近一次操作刚添加的条目，而非按主体选择性移除。当被重置的主体不是最后操作者时，在分桶模式下全局计数器可能错误移除其他主体的条目。需要为重置提供能准确定位特定主体使用量的机制。
2. SubjectNotFoundError 异常类在模块内定义并导出但从未被实际抛出，属于死代码。该异常要么在查询不存在主体时实际抛出来完善 API 语义，要么应从代码中彻底删除。
3. reset_subject 和 reset_global 方法在重置时将全局计数器逐条回滚，每次单独获取释放锁，产生不必要的 O(n) 锁操作开销。应利用回滚方法已有的批量参数一次性完成，避免重复加锁。
4. 修复后为该模块补充对应的测试用例：验证分桶模式下先操作 A 再操作 B 再重置 A 不会影响 B 的计数；验证查询不存在主体时的异常抛出行为。测试代码放在 tests/rate_cap/ 下。
5. 更新 README.md，说明重置操作的并发语义和计数回滚的正确性保证。
6. 重新运行 poetry run pytest tests/rate_cap/ -v，需要让所有测试通过。
