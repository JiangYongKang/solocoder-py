在当前 Python 项目中实现日志段保留与压缩域，所有相关代码放在 src/solocoder_py/log_segment/ 包下，使用内存数据结构模拟日志段存储。
1. 实现按 key 保留最新：日志由多个有序的段（segment）组成，每个日志条目包含 key 和 value。压缩操作遍历全部段，对相同 key 的条目只保留最新的一条（按写入偏移排序），删除旧版本，释放存储空间。
2. 实现过期段回收：每个日志段记录自身的创建时间，可以配置段的保留时间。当段的存活时间超过保留时间后，整个段被标记为可回收状态，后续清理时直接删除整个段文件（该段内所有条目不再可访问）。
3. 实现压缩后的偏移映射：日志压缩完成后，被保留的条目的物理偏移会发生变化，需要维护一份从逻辑偏移到压缩后物理偏移的映射表，使得外部使用者按原始偏移读取时能正确定位到压缩后的新位置。
4. 为该模块编写完整的单元测试，覆盖正常流程（写入多条同 key 条目后压缩保留最新、过期段被回收不可读、偏移映射正确指向压缩后位置）、边界条件（单条记录的压缩、空日志压缩、所有段均过期的回收）和异常分支（压缩过程中有新写入、按不存在的偏移读取、回收已被回收的段）。测试代码放在 tests/log_segment/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、压缩策略与偏移映射机制以及使用示例。
