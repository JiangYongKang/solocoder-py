你实现的超时管理器模块存在一些问题，需要修复：
1. add_callback 方法在传入 None 回调时抛出的是与期限相关的异常类型，但回调参数为空与期限值无效是两种完全不同的错误场景，应使用语义匹配的异常类型传递准确的错误信息。
2. create_child_context 和 cancel_context 方法中，当传入一个根本不存在的上下文 ID 时抛出的是"已取消"类型的异常，但"上下文不存在"和"上下文已取消"是完全不同的概念。需要为上下文不存在定义专门的异常类型。
3. create_child_context 方法对已过期父上下文和已取消父上下文使用了两种不同的异常类型，但两者都是父上下文已达终态不可再创建子上下文的同类约束，异常类型应统一。
4. README.md 声称 is_cancelled 和 is_expired 两种状态可以共存，但代码逻辑确保了两种状态严格互斥。需要修正文档使其准确描述实际的状态互斥行为。
5. 修复上述问题后运行 poetry run pytest tests/timeout_manager/ -v，确保所有测试通过。
