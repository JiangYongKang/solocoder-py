在当前 Python 项目中实现无锁栈（Treiber Stack），基于 CAS 原子操作实现并发安全的压入与弹出。
1. 实现 CAS 原子压入（push）：使用单向链表节点存储栈元素，栈顶指针指向当前栈顶节点。压入新节点时，新节点的 next 指向当前栈顶，然后使用 CAS（Compare-And-Swap）原子操作将栈顶指针从当前值更新为新节点。如果 CAS 失败（说明其他线程同时修改了栈顶），重试直到成功。CAS 操作基于 Python 的原子引用机制或 threading 模块提供的能力。
2. 实现 CAS 原子弹出（pop）：弹出时读取当前栈顶节点，使用 CAS 原子操作将栈顶指针更新为当前栈顶节点的 next。如果 CAS 失败则重试。弹出成功时返回节点的值；栈为空时返回 None 或明确标识空栈。
3. 实现并发安全保证：多线程并发压入和弹出操作在无显式锁的情况下全部正确执行，所有压入的元素最终都能被弹出（无丢失），弹出的元素应唯一（无重复）。使用原子操作保证 ABA 问题的防护（例如使用版本号或标记指针），对内存管理采用与 Python GC 兼容的策略（不依赖手动内存回收）。
4. 提供栈的基本操作：除 push 和 pop 外，提供 peek 方法（查看栈顶元素不移除）、is_empty 方法（判断栈是否为空）、size 方法（返回当前栈中元素数量，需注意 size 在并发场景下为近似值而非精确快照）。
5. 代码实现需要放在 src/solocoder_py/treiber_stack/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（顺序 push pop 正确、空栈 pop 返回 None、peek 返回栈顶不移除、is_empty 判断正确、全部 push 元素均能 pop 且无重复）、边界条件（空栈的 peek 和 pop、单元素栈的 pop 后状态、大量元素 push 后逐一 pop 的正确性、重复 push 相同值多次每份独立存储）和异常分支（并发多线程同时 push 的线程安全（无元素丢失）、并发多线程同时 pop 的线程安全（无重复取出）、并发 push 和 pop 交错执行时的数据一致性、ABA 问题场景的防护验证（线程 A 读取栈顶后线程 B 弹出又压入相同值，线程 A 的 CAS 应正确检测到变化））。测试代码放在 tests/treiber_stack/ 下。
