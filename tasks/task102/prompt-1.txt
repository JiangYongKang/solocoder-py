在当前 Python 项目中实现 DNS 存根解析器，所有相关代码放在 src/solocoder_py/dns/ 包下，使用内存数据结构模拟 DNS 记录缓存与上游递归解析器。
1. 实现 DNS 记录缓存与 TTL 过期：解析器维护一个内存缓存存储解析结果。每条缓存记录带有 TTL（Time-To-Live），缓存命中时检查 TTL 是否过期，过期记录不应返回给调用方。相同域名的不同记录类型（A、AAAA、CNAME 等）独立缓存、独立过期。
2. 实现递归解析委托：解析器收到查询请求后，先检查本地缓存。缓存未命中时委托给上游递归解析器（通过可注入的委托接口），由上游完成完整的递归解析过程并返回最终结果。上层调用方不感知解析是否走缓存还是走委托。
3. 实现 CNAME 链跟踪：当解析结果包含 CNAME 记录时，解析器自动跟踪 CNAME 链直到获取到最终的非 CNAME 记录（如 A 或 AAAA 记录）。CNAME 链的每一级记录同样受到 TTL 控制，中间 CNAME 过期需重新解析。需要防止循环 CNAME 引用导致无限跟踪。
4. 为该模块编写完整的单元测试，覆盖正常流程（首次解析走委托并缓存结果、再次解析命中缓存返回、CNAME 链跟随获取最终 IP、缓存过期后重新委托解析）、边界条件（TTL 正好到期的边界命中判定、空缓存首次查询委托返回空结果、多级 CNAME 链深度跟踪）和异常分支（循环 CNAME 引用检测与终止、上游委托解析超时或返回错误时的处理、缓存中存在不同记录类型的同名记录互不干扰、TTL 设为零的记录不入缓存）。测试代码放在 tests/dns/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、缓存与 TTL 过期机制、CNAME 链跟踪规则以及使用示例。
