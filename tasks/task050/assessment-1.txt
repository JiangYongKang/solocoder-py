产物不满意：
1. registry.py TTL 过期处理设计为手动调用 cleanup_expired() 方法，提示词要求"自动从可用实例列表中摘除"，但当前实现仅在 get_instances 和 select_instance 查询时惰性过滤，get_all_instances 仍返回过期实例，实例数据仍驻留存储不自动释放。应在查询或选择时同步从 _services 中摘除过期实例，或启动后台定时线程定期扫描清理。
2. README 将 cleanup_expired() 描述为"手动清理所有已过期的实例"，文档明确写的是"手动"而非"自动"，与提示词需求自相矛盾，开发者会误认为模块不提供自动摘除能力。应说明摘除的自动触发机制，将 cleanup_expired() 标注为内部优化清理。
3. test_select_instance_mixed_zero_and_positive_weights 对零权重和正权重实例选择分布只用大于关系断言，未断言零权重实例选择次数为 0。当 total_weight > 0 时零权重实例理论上永远不会被选中，但当前断言无法严格验证。应增加 counts["inst-zero-1"] == 0 和 counts["inst-zero-2"] == 0 的断言。
过程不满意：
1. 运行测试时使用 python -m pytest 实际运行了 watchdog 模块的测试而非 registry 模块，浪费一次工具调用和推理时间。应始终使用 poetry run pytest 执行测试，并确保指定正确的测试目标目录。
