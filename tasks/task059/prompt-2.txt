你实现的发布订阅模块存在一些问题，需要在 "src/solocoder_py/pubsub/" 范围内修复：
1. 慢订阅者触发 "DROP_OLDEST" 背压策略时，丢弃记录会写到新消息上，而真正被挤掉的旧消息没有准确投递状态。
2. 公开的 "BLOCK" 背压策略没有真实阻塞或限流行为，缓冲区满后仍会继续追加消息，可能导致慢订阅者无限堆积。
3. 背压相关测试只检查状态存在或缓冲区长度，没有验证每条消息对每个订阅者的最终投递记录是否准确。
4. 补充测试覆盖被丢弃消息的 message_id、同一消息不能同时成功和丢弃、以及 "BLOCK" 策略在缓冲区满时的明确行为。
5. 更新 README 中背压策略和投递状态追踪说明，并运行 "poetry run pytest tests/pubsub/ -q"。
