在当前项目四叉树空间索引的实现中，仍存在以下问题需要修复：

1. quadtree.py 中 Quadtree.insert_rectangle 方法将越界校验完全委托给内部 _QuadNode，自身没有在公共 API 入口做前置校验。当用户插入的矩形超出四叉树边界时，越界错误从深层递归的 _QuadNode 内部抛出，异常消息引用的是某个分裂子象限的局部边界坐标而非用户传入的四叉树整体边界，用户难以将错误信息对应到原始输入参数。
2. 上次提示词中问题 2（边界对象处理不一致）的代码修复已应用，但缺少对应的测试验证。现有 test_clear_preserves_configuration 测试只校验了 clear 后边界 width 和 height 的数值相等，没有验证 qt.boundary 和 qt._root.boundary 是否为同一实例，也没有验证 clear 后重建的根节点边界来源与初始构造时是否一致。如果未来有人回退了这处修复，当前的测试套件不会产生任何失败信号。
