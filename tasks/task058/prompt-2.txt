你实现的 Webhook 投递模块存在一些问题，需要在 "src/solocoder_py/webhook_delivery/" 范围内修复：
1. 单条投递入口没有检查下一次可投递时间，失败后即使还没到指数退避时间，也可以被外部再次立即投递。
2. 重试次数到达上限时转入死信队列的行为和 README 描述不一致，使用者无法准确理解 "max_retries" 的边界含义。
3. 签名失败测试同时修改 payload 和密钥，无法证明载荷被篡改时签名校验本身会失败。
4. 补充测试覆盖未到重试时间不能直接投递、最大重试次数边界、以及只篡改 payload 时签名校验失败的场景。
5. 同步更新 README 中重试、死信和签名校验语义，并运行 "poetry run pytest tests/webhook_delivery/ -q"。
