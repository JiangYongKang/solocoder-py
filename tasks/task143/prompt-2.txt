在当前项目中修复 tag_hierarchy 层级标签系统的一些问题：
1. cleanup_dangling_tags 方法在已持有锁的状态下调用了 find_dangling_tags，后者内部也会尝试获取同一把锁。当前使用可重入锁暂时避免了死锁，但这种持锁方法调用另一个持锁方法的设计让锁的边界模糊，如果将来锁类型改为普通 Lock 会直接死锁。应让锁的获取和释放只发生在外层公有方法中。
2. find_objects_by_tags 的 README 文档宣称"效率优于逐标签查询后再求交集"，但实际实现仍是先为每个标签完整计算对象集合再逐集合求交，排序和提前终止没有改变核心流程的复杂度，与朴素方案属于同一数量级。需要考虑从数据结构层面做真正的优化。
3. object_has_tag 方法对对象的每个直接标签都单独做一次完整祖先链遍历，当对象被标记多个深度嵌套标签时同一个祖先节点被重复遍历多次。而 get_object_tags 方法已经实现了更高效的做法——先收集所有祖先再统一去重。需要让 object_has_tag 复用同样的高效策略。
4. 更新 README.md，如果交集查询的效率未从数据结构层面优化，则修正文档中的过度声称；同步更新其他修改点的说明。
5. 修复后重新运行 poetry run pytest tests/tag_hierarchy/ -q，确保所有测试通过。
