你实现的灰度发布模块存在一些问题，需要在 "src/solocoder_py/canary/" 范围内修复：
1. 多个发布同时存在时会共用同一个路由状态，后创建的发布会覆盖先前发布的基线版本、候选版本和流量比例，导致旧发布请求被路由到错误版本。
2. 手动回滚或指标触发回滚时，回滚记录中的流量比例被写成 0，无法保留回滚发生前的真实放量比例。
3. 请求指标记录没有和实际路由结果关联，基线请求和候选请求可能被混在候选指标里，影响错误率和延迟判断。
4. 补充测试覆盖多个发布同时路由、不同放量比例下的回滚审计记录、以及基线和候选指标分离统计的场景，并更新 README 中相关说明。
5. 修复后运行 "poetry run pytest tests/canary/ -q"，确保该模块测试通过。
