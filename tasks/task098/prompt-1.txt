在当前 Python 项目中实现三方文本合并域，使用内存数据结构模拟文本行与合并操作。
1. 实现基于共同祖先的无冲突合并：给定三份文本 —— 共同祖先版本（base）、版本 A 和版本 B，系统需要将 A 和 B 的修改合并到 base 之上生成最终合并结果。如果 A 和 B 修改的是不同区域（行），都应被采纳到最终结果中。如果 A 和 B 对同一区域做了完全相同的修改，只采纳一次。
2. 实现冲突标记：当 A 和 B 对同一区域（同一行或同一行块）做了不同的修改时，合并结果应在该区域插入标准的冲突标记，清晰标注出 A 方的修改内容和 B 方的修改内容，保留双方版本供人工裁决。其他无冲突区域保持正常合并。
3. 实现块级对齐：合并时不应按逐行字符比较，而应将文本划分为逻辑块（如连续的变更行组成的块），对块进行对齐后再比较。块级对齐能更好地识别出移动、插入和删除操作，减少因简单行号偏移导致的大量误报冲突。
4. 代码实现需要放在 src/solocoder_py/three_way_merge/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（A 新增行 B 未修改的无冲突合并、A 修改行 B 删除另一行的无冲突合并、AB 做了完全相同修改的去重合并）、边界条件（A 或 B 等于 base 无需合并、三份文本完全相同的零冲突合并、base 为空的合并）和异常分支（AB 修改同一行但内容不同的冲突标记生成、AB 分别插入到同一位置相邻行的正确排序、大片区域修改的块级对齐精确性）。测试代码放在 tests/three_way_merge/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、三方合并算法的逻辑、冲突标记格式与块级对齐策略以及使用示例。
