在当前 Python 项目中实现 JSON Patch (RFC 6902) 引擎，使用内存数据结构操作 JSON 文档。
1. 实现六种原子操作：按照 RFC 6902 规范实现以下操作。add：在指定路径添加新值，路径指向数组末尾时使用 "-" 表示追加，路径指向对象不存在键时创建键值对；remove：删除指定路径的值，如果路径指向的父对象或索引不存在则报错；replace：将指定路径的现有值替换为新值，语义等价于先 remove 再 add；copy：将 from 路径的值复制到 path 路径，如果 from 路径不存在则报错；move：将 from 路径的值移动到 path 路径，语义等价于先 copy 再 remove from；test：检查指定路径的值是否等于预期值，相等则通过，不等则报错（整批次操作失败）。每个操作的路径使用 JSON Pointer (RFC 6901) 格式（以 / 分隔的路径段，支持 / 转义和数字索引）。
2. 实现原子操作序列：接收一个操作列表，按顺序应用全部操作。如果任意一个操作失败（test 不通过、路径不存在、路径格式错误等），整个序列回滚，JSON 文档恢复到应用前的状态。操作序列要么全部成功，要么全部不生效。
3. 实现 JSON Pointer 路径解析：提供独立的 JSON Pointer 解析和求值工具。支持通过 JSON Pointer 字符串在 JSON 文档中定位值、设置值、删除值。正确转义 `~0`（表示 `~`）和 `~1`（表示 `/`）。
4. 代码实现需要放在 src/solocoder_py/json_patch/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（六个操作的独立正确性验证、操作序列的完整应用、JSON Pointer 路径定位嵌套对象和数组、copy/move 跨路径操作的正确性、test 操作通过和不通过的情况）、边界条件（空操作列表的应用、路径指向根 "/"、路径指向数组越界索引的 add/remove、"\\x2d"  作为数组最后元素的 add、深层嵌套的路径操作、多层 / 转义的正确处理）和异常分支（test 失败导致序列回滚、不存在路径的 remove 报错、不存在路径的 replace 报错、非法 JSON Pointer 格式拒绝、操作列表中包含未知操作类型的报错、from 路径不存在的 copy/move 报错、add 操作在数组中插入越界索引的处理）。测试代码放在 tests/json_patch/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、RFC 6902 六种操作的定义与示例、RFC 6901 JSON Pointer 路径语法、原子序列应用与回滚机制以及使用示例。
