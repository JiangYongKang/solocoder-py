在当前 Python 项目中实现设备影子（数字孪生），使用内存数据结构存储设备状态。
1. 实现预期态与上报态的双态模型：每个设备影子维护两个状态副本：desired（预期态，由云端应用设置，表示期望设备达到的目标状态）和 reported（上报态，由设备上报，表示设备当前的实际状态）。两个状态各自独立存储为 JSON 对象。提供分别设置 desired 和 reported 的接口。
2. 实现预期态与上报态的合并：提供将 desired 和 reported 合并为完整状态的方法。当同一字段在两个状态中都存在时，以 reported 为准（设备实际值）。在 desired 中存在但 reported 中不存在的字段表示尚未达成的期望，同样纳入合并结果。
3. 实现差异计算：对比 desired 和 reported，输出两部分的差异：仅存在于 desired 中的字段（期望但未达成）、仅存在于 reported 中的字段（设备实际有但未期望）、相同字段但值不同的差异列表。差异以结构化数据输出（字段路径和两边的值）。
4. 实现带版本同步：每个设备影子维护一个单调递增的版本号。每次对 desired 或 reported 的更新操作时，调用方可传入一个预期的当前版本号。如果传入的版本号与影子当前版本不一致（说明中途发生了并发修改），则拒绝本次更新（乐观锁），防止并发写入造成状态丢失。
5. 代码实现需要放在 src/solocoder_py/device_shadow/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（设置 desired 和 reported、合并状态正确、差异计算准确、版本号递增、同一版本号连续更新的冲突拒绝）、边界条件（初始版本号为 1 或 0、desired 和 reported 均为空对象、二者完全一致时差异为空、深层嵌套 JSON 的差异比对、仅 desired 无 reported 的合并、仅 reported 无 desired 的合并）和异常分支（版本号不匹配的更新拒绝、版本号传入负数的处理、非 JSON 可序列化对象的设置拒绝、传入空值或 None 的 desired/reported 处理）。测试代码放在 tests/device_shadow/ 下。
6. 在该模块包内编写 README.md，说明模块功能、核心类的职责、双态模型的语义、差异计算规则、乐观锁版本同步机制以及使用示例。
