你实现的 Semver 语义版本解析比较器存在一些缺陷需要修复：

1. SemverVersion 构造函数对 prerelease 参数没有格式校验。parse 方法会拒绝 "1.0.0-01" 这类前置零预发布标识符，但直接调用 SemverVersion(1, 0, 0, prerelease="01") 却能成功创建对象，两条 API 路径对同一非法输入的行为不一致。构造函数也需要做同样的格式校验。

2. parse 方法接受了 "1" 这样的单数字简约形式并解析为 1.0.0，但 Semver 2.0 规范要求正常版本号必须采用 X.Y.Z 三部分完整形式。当前实现偏离了规范要求，需要让这类输入走非法版本字符串的异常路径。

3. 重新运行 poetry run pytest tests/semver/ -v 验证修复后所有测试仍然通过。
