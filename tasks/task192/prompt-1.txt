在当前 Python 项目中实现设备注册与证书签发域，使用内存数据结构存储设备信息和证书。
1. 实现预共享密钥注册：设备使用预共享密钥（PSK）向注册中心发起注册。注册时需提供设备标识和预共享密钥，注册中心校验密钥是否与预设的密钥列表匹配。校验通过后生成设备唯一 ID 并记录设备信息（标识、注册时间、状态）。同一设备标识不能重复注册。
2. 实现 CSR 提交：已注册设备可以提交证书签名请求（CSR），内容包括设备 ID 和公钥信息。注册中心收到 CSR 后验证设备 ID 是否存在且状态为已注册，验证通过后使用 CA 证书签发设备证书。证书需包含序列号、颁发者、主题（设备 ID）、有效期和公钥信息。
3. 实现证书吊销：提供证书吊销接口，输入证书序列号或设备 ID 将对应证书标记为已吊销。已吊销的证书在后续查询时应返回吊销状态。提供按设备 ID 查询其所有证书（含已吊销）和按证书序列号查询单张证书详情的接口。
4. 代码实现需要放在 src/solocoder_py/device_cert/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（PSK 校验通过后注册成功、注册后提交 CSR 并成功签发证书、吊销证书后状态变更正确、按设备 ID 查询所有证书、按序列号查询单张证书）、边界条件（已吊销设备再次提交 CSR 的处理、证书有效期的边界值、同一设备提交多个 CSR 签发多张证书）和异常分支（错误 PSK 的注册拒绝、未注册设备提交 CSR 的拒绝、重复设备标识的注册拒绝、吊销不存在的证书序列号的处理、CSR 中设备 ID 不存在时的拒绝、已吊销证书再次吊销的幂等处理）。测试代码放在 tests/device_cert/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、设备注册流程（PSK 校验 → 生成设备 ID → 注册完成）、CSR 签发流程、证书吊销机制以及使用示例。
