你在上一轮修复请求体大小限制器时引入了两个新问题，需要修复：

1. limiter.py 中 _build_internal_error_response 方法将异常对象的完整字符串表示（str(exc)）直接写入 HTTP 响应体的 message 字段。当异常对象携带内部调用栈或文件路径等敏感信息时，这些信息会通过响应体直接泄露给客户端。修复方式不应是把异常详情暴露在响应体中，应使用固定的通用错误提示，避免信息泄露。

2. limiter.py 的 _from_sequence 方法采用先将序列中所有元素拼接为一个完整缓冲区再分块的方式修复 chunk_size 被忽略的问题，丢失了原实现按元素逐个迭代的流式特性。当序列总大小远超限制时，所有数据会被一次性全部加载到内存中造成不必要的内存开销。需要在迭代序列元素时以 chunk_size 为窗口逐段切割每个元素输出，而非先整体拼接再分块，保持流式读取的特性。

3. 修复以上问题后重新运行 poetry run pytest tests/request_limiter/ -q，确保所有测试仍然通过。
