产物不满意：
1. _split_sentences 里英文句尾分割正则要求后续字符必须是大写字母 (?=[A-Z])，但英文句子后紧跟中文比如"Machine learning is important. 机器学习非常重要。"时，中文"机"不满足大写要求，正则匹配不上，中英混合文本被当成一个句子。词频统计把两句混算，句子打分就歪了。根因是正则没覆盖英语后面接非大写字符的场景，应该在分割逻辑里增加处理。
2. SummarizerConfig 的 position_weight_factor 默认值是 1.0，配合线性衰减公式 weight = factor + distance * (1.0 - factor)，当 factor=1.0 时计算结果永远是 1.0，所有句子位置权重一样。这导致默认配置下 LINEAR 衰减跟 NONE 完全等价，提示词要求"开头和结尾句子给更高权重"在默认参数下根本不生效。应该把默认值改成 0.5 之类的值，让默认线性衰减真的产生差异。
3. README 核心类职责章节把 summarize_text 便捷函数错描述成一个叫 Summarizer 的类，但 __init__.py 实际导出的只有函数 summarize_text，代码里根本不存在 Summarizer 这个类。用户按 README 用会困惑。根因是写 README 时没跟 __init__.py 的实际导出内容做交叉核对。

过程不满意：
1. 模型两次跑测试都用了 python -m pytest 而不是 poetry run pytest，虽然这次虚拟环境恰好激活没出事，但这绕过了 pyproject.toml 定义的依赖管理和测试配置。根因是没在写测试前看 pyproject.toml 了解测试运行约定。
2. 模型首版 summarizer.py 有两处核心算法错：位置权重算反了（中间句子权重更高），以及中文标点分割正则不完整。首轮测试出 3 个失败才回头改，属于一轮试错修复。根因是写完核心逻辑后没先用简单样例做笔算验证就直接跑测试，应该在编码阶段用笔算检查几个典型场景再进测试。
