你实现的布隆过滤器域功能模块存在一些问题，需要修复：
1. CountingBloomFilter 的 remove 方法在抛出 ValueError 时报错消息字符串漏写了 f-string 前缀，导致占位符以字面文本出现在异常信息中，用户无法获知是哪个索引位置计数器为零导致删除失败。
2. BloomFilter 和 CountingBloomFilter 两个变体中的哈希索引计算、序列化方法和 FNV-1a 哈希函数代码完全重复。未来更改哈希算法或序列化逻辑需要同步维护两处，容易遗漏导致两个变体行为不一致。应提取共享逻辑消除重复。
3. 过滤器的并集和交集操作中 count 属性的估算值仅取双方 count 的 max 或 min。当两个过滤器元素集合完全不重叠时，并集 count 应接近两者之和，但当前仅取了最大值。需要在文档中说明 count 估算的局限性并在可能的情况下给出更合理的上界。
4. 修复后为该模块补充对应测试用例：验证 remove 报错消息中索引值正确替换；验证并集 count 在不重叠场景下的估算合理性。测试代码放在 tests/bloom_filter/ 下。
