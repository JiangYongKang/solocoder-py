在当前 Python 项目中实现 Markdown 转 HTML 转换器域，使用内存数据结构进行文本处理。
1. 实现 Markdown 核心语法转 HTML：支持标题（h1-h6）、段落、加粗、斜体、行内代码与代码块、有序和无序列表、链接、图片、引用块和水平分割线的转换。
2. 实现表格转换：支持 Markdown 表格语法的解析和转换，包括表头行、分隔行、数据行，输出符合 HTML 标准的 table/thead/tbody/tr/th/td 结构。
3. 实现代码高亮钩子：代码块转换时提供可扩展的钩子机制，允许外部注入语言对应的高亮函数。如果未注册对应语言的钩子，代码块应正常输出但不高亮。系统需提供一个内置的钩子注册表和默认钩子示例。
4. 实现安全 HTML 清洗：转换后的 HTML 输出需要进行安全清洗，移除或转义危险的标签和属性（如 script、onerror 等事件属性），防止 XSS 注入。Markdown 源文中已经存在的原始 HTML 标签也需要清洗处理。
5. 代码实现需要放在 src/solocoder_py/markdown_html/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（各种语法的单元素转换、复合文档整体转换）、边界条件（空输入、嵌套列表、表格内包含行内格式、代码块中包含 Markdown 语法字符）和异常分支（包含 script 标签被清洗、包含事件属性被移除、表格列数不一致时的容错处理、未闭合的代码块处理）。测试代码放在 tests/markdown_html/ 下。
6. 在该模块包内编写 README.md，说明模块功能、核心类的职责、支持的语法列表、钩子扩展机制（可用文本描述）以及使用示例。
