在当前 Python 项目中实现 TOML/INI 配置解析器，使用内存数据结构存储解析后的配置树。
1. 实现 TOML 格式解析：支持 TOML v1.0 规范的核心语法，包括键值对、字符串（基本字符串和字面量字符串）、整数、浮点数、布尔值、日期时间（本地日期时间和带偏移的日期时间）、数组、行内表、标准表（`[table]`）和数组表（`[[array]]`）。表支持嵌套定义（如 `[a.b.c]`）。
2. 实现 INI 格式的兼容解析：作为 TOML 的超集兼容解析简化的 INI 格式。INI 格式支持 `[section]` 段分隔、`key=value` 键值对、以 `;` 或 `#` 开头的注释行。同一个段内的同名键后者覆盖前者。
3. 实现类型强制转换：提供便捷方法将已解析的配置值按指定类型获取，支持：bool（"true"/"false" 及大小写变体、1/0 数字）、int（十进制整数）、float（浮点数）、array（数组，元素类型需一致）。如果值无法转换为目标类型则报错。
4. 实现注释保留：解析配置后，每个键值对应保留其上方最近的非空注释行（以 `#` 开头）。通过注释查询方法可以获取每个键关联的注释文本。
5. 代码实现需要放在 src/solocoder_py/config_parser/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（完整 TOML 文档的键值解析、嵌套表结构的树形遍历、数组表的元素列表解析、INI 格式的段与键值解析、类型转换的正确性、注释提取）、边界条件（空配置文档、仅含注释的文档、深层嵌套表的遍历、数组元素为空时的处理、超大整数的解析）和异常分支（语法错误的 TOML 文档报错、键重复定义的处理、类型转换失败的错误信息、非法日期格式的处理、未闭合字符串的报错、段名格式不合法的拒绝）。测试代码放在 tests/config_parser/ 下。
6. 在该模块包内编写 README.md，说明模块功能、核心类的职责、TOML/INI 格式规范简述、类型转换规则表、以及使用示例。
