在当前 Python 项目中实现订单结算域，所有相关代码放在 src/solocoder_py/order/ 包下，使用内存数据结构模拟数据源。
1. 实现订单状态机，包含以下状态和合法转移路径：待支付 -> 已支付 -> 已发货 -> 已签收 -> 已完成，以及取消（待支付状态下可取消）、退款（已支付后可申请退款）、售后（已签收后可发起售后）等分支流转。状态转移需要做合法性校验，不允许非法跳转。
2. 部分履约场景下，订单可以分多次发货，每次发货更新对应行项目（OrderLineItem）的履约数量，直到全部行项目均履约完成时才允许订单进入已签收状态。需要记录每次发货的明细。
3. 支持促销叠加规则：多个促销活动可以同时应用于一个订单，但需要校验互斥规则（如满减与直减不能同时生效，折扣与特价不能同时生效），同时计算叠加后的最终金额。每个促销活动需记录其类型（满减/直减/折扣/特价）和适用条件。
4. 为该模块编写完整的单元测试，覆盖正常流程（订单全生命周期、多次发货、促销叠加计算）、边界条件（空订单、行项目履约数量为零或超额、最大促销数量叠加）和异常分支（非法状态转移、互斥促销同时应用、已取消订单再支付）。测试代码放在 tests/order/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、状态机图（可用文本描述）以及使用示例。
