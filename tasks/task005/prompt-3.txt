令牌模块的 USED 状态防御实现得过于宽泛，需要收窄一下：
1. validate_access_token 新增的防御用了 status != ACTIVE 通吃所有非活跃状态。一个 token 首次校验过期时被正常标为 EXPIRED 并抛 TokenExpiredError，如果再有代码对该 token 做第二次校验，错误类型会从 TokenExpiredError 变成 TokenRevokedError（因为进入了 status != ACTIVE 分支），改变了原有错误路径的语义。把防御改成只检查 status == USED，不要干扰其他状态对应的错误类型。
2. 修复后重新运行测试，需要让所有测试通过。
