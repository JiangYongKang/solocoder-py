产物不满意：
1. 模型在 authority 属性修复后没为这个场景加回归测试。现有测试只断言了有 host 场景下的非 None 返回值，从没对 authority is None 做过断言，None 语义完全没有自动化保护。以后任何人重构这个属性空字符串 BUG 都可能重现。应该加一条针对 file:///path/to/file 的回归测试验证 authority is None。

过程不满意：
1. 模型收到提示词后第一步就创建了 2 项子任务计划，然后才读 parser.py 源码。这和 assessment-2 已指出的问题完全重复——在未理解代码前就定计划。对于一个只需改一行 return 的简单修复，两步计划在未读源码前就制定完成毫无价值。应该先读代码确认修改只影响 authority 不波及 rebuild，再规划具体步骤。

2. 模型编辑完 authority 后，先用 python -c 内联脚本手动验证了三种 URL 的返回值，又创建了临时文件执行完全相同的验证再删掉临时文件。两轮手动验证是明显的徒劳操作，且跟 assessment-2 指出的问题重复出现还变本加厉验证了两遍。prompt 明确要求跑 pytest 验证，直接运行即可确认正确性无需写临时脚本。
