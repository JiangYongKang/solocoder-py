在当前 Python 项目中实现 JWT 签发与校验域，所有相关代码放在 src/solocoder_py/jwt/ 包下，使用内存数据结构存储签名密钥与配置。
1. 实现密钥轮换：JWT 的签名密钥支持多版本并存，每个密钥有唯一标识（kid）。签发 JWT 时使用当前活跃的密钥，并在 JWT 头部写入 kid 字段标识使用的密钥。校验 JWT 时，根据头部 kid 找到对应的密钥进行签名验证。密钥轮换时新增一个密钥并将其设为活跃，旧密钥保留一段时间用于校验已签发的旧 JWT，超期后从密钥集中移除。轮换过程中已签发的旧 JWT 不应因密钥切换而立即失效。
2. 实现算法白名单：JWT 支持多种签名算法（如 HS256、HS384、HS512 等 HMAC 算法）。签发和校验时必须校验所用的算法是否在配置的算法白名单中，白名单之外的算法（如 none 算法）应被明确拒绝。在签发时，白名单控制可以使用哪些算法生成签名；在校验时，白名单控制接受哪些算法的 JWT。JWT 头部的 alg 字段与白名单不匹配时直接判定无效。
3. 实现 aud/iss 声明强制校验：JWT 的载荷中包含标准声明字段。其中 iss（签发者）声明在校验时需要与预期的合法签发者列表比对，不在列表中的 iss 值应判定为无效。aud（受众）声明表示该 JWT 的目标接收方，校验时需要检查当前服务是否在 aud 列表中，如果 aud 列表不包含当前服务则判定为无效。这两个字段的校验是强制开启的，不可跳过。
4. 为该模块编写完整的单元测试，覆盖正常流程（HMAC 算法签发后成功校验、密钥轮换后新旧密钥各自签发的 JWT 都能通过校验、算法白名单内算法通过白名单外被拒绝、aud 和 iss 校验通过、过期时间 exp 校验生效）、边界条件（颁发时间为过去或未来的 nbf 声明、签发时间与校验时间的边界、aud 为数组包含多个值时当前服务命中任意一个即通过、kid 不存在对应密钥时校验失败、密钥集为空时签发报错）和异常分支（none 算法被白名单拒绝、使用已移除的旧密钥签名的新 JWT 校验失败、签名被篡改的 JWT 校验失败、JWT 格式不正确（缺少分隔点）的解析报错、载荷不是合法 JSON 的 JWT 解析失败、exp 过期 JWT 被拒绝且不能通过其他声明绕过）。测试代码放在 tests/jwt/ 下。
