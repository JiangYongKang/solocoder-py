你实现的 JWT 签发与校验模块存在一些问题，需要修复：
1. aud 校验的 `_verify_aud` 方法在校验方未显式传入当前服务身份时，会自动回退使用配置中 audiences 列表的第一个元素作为期望值。由于签发端始终将完整的 audiences 列表写入 JWT 的 aud 字段，这导致期望值必然出现在 aud 列表中，aud 校验变成了自验证循环，完全丧失了跨服务受众校验的安全意义。校验方必须显式声明自己的服务身份，当未提供当前服务身份时应拒绝校验而非使用回退值。
2. `encode` 方法在 `expire_seconds` 为 0 时不会向 JWT 载荷写入 exp 字段，而 `decode` 流程中的 `_verify_exp` 方法强制要求 exp 字段必须存在。这导致通过合法 API 签发的 JWT 在校验时必定失败，签发端和校验端对 exp 的必选性约定不一致。
修复后重新运行 poetry run pytest tests/jwt/ -q，需要让所有测试通过。
