2. validator.py 的 _check_validity 方法将有效期比较从原始浮点值改为 int 截断后引入了新的精度边界 bug。当 not_after 为 1000.0 且 now 为 1000.9 时，int 截断后两者均为 1000，now_sec 大于 not_after_sec 条件不成立，证书实际已过期却被判定为有效。同理 not_before 为 1000.9 且 now 为 1000.1 时，证书尚未到达生效时间却被误判为有效。根因是 int 向下截断丢失了亚秒信息，导致同秒内的临界时间窗口被错误忽略。正确做法是在保持浮点精度比较的基础上统一签名构造，或者用 math.ceil 对 not_before 向上取整、math.floor 对 not_after 向下取整以确保截断方向与语义一致。由于所有测试用例均使用整数时间戳，该 bug 未被现有 40 个测试覆盖。
3. validator.py 的 _check_validity 方法在将时间比较改为 int 截断后，抛出的 CertificateExpiredError 和 CertificateNotYetValidError 异常构造参数仍使用原始浮点值 cert.not_before、cert.not_after 和 now，但实际判断逻辑使用的是 int 截断后的值。当因截断逻辑触发异常时，异常消息中显示的时间值与内部判定依据的截断值不一致，例如 not_after 为 1000.9、now 为 1001.0 时，int(now)=1001 大于 int(not_after)=1000 触发过期，但异常携带的是原始浮点值 1000.9 和 1001.0，虽在此特定场景下数值方向一致，但在更复杂的亚秒边界场景下这种不一致会给调用方的诊断和日志分析带来混淆。应在异常构造中统一传入实际用于决策的截断后时间值或直接恢复浮点比较。
