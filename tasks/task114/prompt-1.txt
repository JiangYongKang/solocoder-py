在当前 Python 项目中实现凭据轮换编排器，所有相关代码放在 src/solocoder_py/credential/ 包下，使用内存数据结构存储凭据版本与切流状态。
1. 实现双写过渡期：凭据轮换过程中存在新旧两套凭据。编排器在过渡期内将写操作同时发送到新旧两套凭据对应的目标系统（如同时写入新旧两个密钥到配置中心），确保在切换完成前两套凭据都可用。双写期间如果任一写入失败，需要记录失败信息但不应中断另一侧的正常写入。过渡期的持续时间可配置。
2. 实现灰度切流：凭据使用者从旧凭据迁移到新凭据不应是一次性全量切换，而是按百分比逐步放量。编排器维护一个切流比例（0% 到 100%），根据请求中的标识（如请求 ID 或主体 ID 的哈希值）决定该请求使用旧凭据还是新凭据。切流比例逐步递增直到 100%，每次递增步长可配置。灰度过程中如果新凭据的请求错误率超过阈值，自动暂停切流甚至回退比例。
3. 实现失败自动回退到旧凭据：使用新凭据的请求如果连续失败次数达到阈值，编排器自动将读操作回退到旧凭据。回退后的退避期不可立即再次尝试新凭据，需等待一段冷却时间后才能再次逐步恢复切流。回退事件需要被记录，包含回退时间、触发回退的失败次数和切流当时的比例。
4. 为该模块编写完整的单元测试，覆盖正常流程（凭据从旧到新的完整灰度切换流程、双写期间新旧凭据均可正常使用、切流比例按步长递增直到 100%、过渡期结束后确认新凭据为唯一有效凭据）、边界条件（切流比例 0% 时全部使用旧凭据、切流比例 100% 时全部使用新凭据、双写中旧系统写入失败不影响新系统、灰度过程中错误率刚好达到阈值边界）和异常分支（新凭据连续失败触发自动回退到旧凭据、回退冷却期内禁止重新切流、双写全部失败时的告警记录、切流比例被手动回退到之前更低比例的处理、冷却期内新凭据恢复正常后自动重新开始灰度、凭据轮换过程中编排器自身崩溃重启后的状态恢复）。测试代码放在 tests/credential/ 下。
