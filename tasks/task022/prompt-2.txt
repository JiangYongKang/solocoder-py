wal 包里有几处可以改进的地方，在现有代码上改一改。
1. replay 方法用 for 循环把所有条目收集到 list 再返回 iter(entries)，一次性物化全量结果。WAL 重放可能涉及大量条目，不应该全量加载到内存里，应该改成惰性生成器按需产出。
2. replay 在遍历 LSN 范围时遇到条目缺失直接用 break 静默停止，不报错也不跳过。虽然当前代码保证 LSN 总是连续，但这种静默停止的设计假设太脆弱，一旦未来引入非连续删除就会导致数据丢失不被发现。遇到不存在的 LSN 应该抛出明确错误而不是悄悄停掉。
3. 修完跑 poetry run pytest tests/wal/ -q，确保全过。
4. README 同步更新，说明 replay 的惰性语义和 LSN 连续性的错误处理策略。
