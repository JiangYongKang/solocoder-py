你实现的 RBAC 模块存在一些问题，需要在 "src/solocoder_py/rbac/" 范围内修复：
1. 权限匹配时动作字段不应使用通配授权，当前 "action='*'" 会扩大成任意操作权限，和只要求资源标识支持通配的需求不一致。
2. 角色对象可以通过创建或查询接口拿到内部可变对象后被外部直接修改，绕过父角色存在性校验和循环继承校验。
3. 深层角色继承依赖递归遍历，层级较深时可能触发递归深度问题，导致权限校验不可用。
4. 补充测试覆盖动作字段不能被通配、外部不能绕过引擎修改角色继承关系、以及更深层级角色继承仍能稳定校验的场景。
5. 更新 README 中权限匹配和角色继承约束说明，并运行 "poetry run pytest tests/rbac/ -q"。
