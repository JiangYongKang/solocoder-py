你实现的通知扇出模块存在一些问题，需要在 "src/solocoder_py/notification_fanout/" 范围内修复：
1. 渠道投递超时后仍会等待底层线程执行结束，导致整体扇出结果不能在渠道超时时间到达后及时返回。
2. 多渠道投递会受到全局 worker 数限制而排队，某些配置下慢渠道会阻塞其他渠道，和各渠道并行且互不阻塞的要求不一致。
3. 注册或更新渠道配置时没有校验配置中的渠道名和实际注册渠道一致，错误配置会静默生效并造成策略归属混乱。
4. 补充测试覆盖永久阻塞或长耗时渠道的超时返回、多渠道在 worker 数不足时的并行语义、以及渠道配置名不匹配应被拒绝的场景。
5. 更新 README 中超时、并行投递和渠道配置约束说明，并运行 "poetry run pytest tests/notification_fanout/ -q"。
