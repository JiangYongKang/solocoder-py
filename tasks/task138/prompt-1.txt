在当前项目中实现一个 API 客户端限流器功能模块，使用内存数据结构模拟数据源。
1. 实现令牌桶算法：维护一个容量有限的令牌桶，令牌以固定速率持续补充直至桶满。每次 API 调用从桶中取走一个令牌，令牌不足时请求被拒绝或等待。桶的容量（最大令牌数）和令牌填充速率均可配置，容量大于补充速率时允许短时间内的突发调用。
2. 实现响应头限额状态透传：模拟上游 API 服务返回的限流响应头（如 X-RateLimit-Remaining、X-RateLimit-Reset），客户端限流器在每次调用后解析这些响应头，根据服务端实际限额状态同步调整本地令牌桶，避免本地估计与服务端实际限额偏差过大。
3. 支持突发余量：令牌桶允许配置突发容量（burst size），即桶中可积累的最大令牌数。当客户端在空闲一段时间后，桶中令牌积累到最大值，此时可在短时间内发出超出平均速率的突发请求，但不能超过桶容量。突发容量耗尽后恢复为匀速限流。
4. 代码实现需要放在 src/solocoder_py/rate_limiter/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程、边界条件（如桶空时请求的行为、桶满后令牌不再增长的验证、突发请求恰好等于桶容量）和异常分支（如服务端返回的 Reset 时间早于当前时间、本地令牌数超过服务端剩余量的同步处理、并发取令牌的线程安全性）。测试代码放在 tests/rate_limiter/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、令牌桶与漏桶算法的区别、响应头透传的作用、突发容量与平均速率的配置关系以及使用示例。
