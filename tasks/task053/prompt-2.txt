你实现的配置热更新模块存在一些问题，需要在 "src/solocoder_py/config_hot_reload/" 范围内修复：
1. 回滚历史版本时不应把回滚内容发布成全新的版本，而是要让被指定的历史版本重新成为当前生效版本。
2. 成功发布或回滚时，即使配置内容没有键级差异，也应让监听器获得版本事件，避免调用方无法感知发布或回滚动作。
3. 单键读取配置时返回的可变对象会泄漏内部快照，调用方可以绕过发布流程原地修改当前配置。
4. 补充测试覆盖回滚后当前版本号、空配置或重复配置发布通知、回滚通知、以及通过 "get" 读取嵌套配置后外部修改不会污染内部快照。
5. 更新 README 中关于回滚语义、监听事件和配置快照隔离的说明，并运行 "poetry run pytest tests/config_hot_reload/ -q"。
