1. 普通 "DENY" 策略被默认当成显式拒绝处理，导致它会绕过允许优先或最高优先级等冲突裁决规则，拒绝效果和显式拒绝语义需要区分。
2. "FIRST_APPLICABLE" 冲突裁决无法稳定表达真正的首个命中策略，现有测试也没有验证最终裁决结果和首个策略身份。
3. README 内容不完整，缺少核心类职责、策略求值顺序、冲突裁决规则和使用示例，无法说明显式拒绝与普通拒绝的差异。
4. 补充测试覆盖普通拒绝和显式拒绝的差异、首个适用策略的最终决策、以及多策略冲突下的审计信息。
5. 补全 README 后运行 "poetry run pytest tests/abac/ -q"，确保该模块测试通过。
