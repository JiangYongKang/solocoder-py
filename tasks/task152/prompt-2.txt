你实现的 Unicode 分词器模块存在一些问题，需要修复：
1. CJK_RANGES 的范围定义过度包含了不属于统一表意文字的 Unicode 区块，如 CJK 符号和标点区、部首增补区、笔画区、表意文字描述字符区等。其中部分区间在多个范围列表（CJK、标点、空白）中重叠定义，虽然当前检测顺序掩盖了冲突但缺乏语义严谨性。
2. exceptions.py 中定义了 UnsupportedScriptError 异常类并在公开 API 中导出，但全模块没有任何地方实际抛出该异常，属于死代码。
3. 英文缩写如 "don't" 被撇号切分为三个独立 Token，丢失了缩写语义。虽然提示词未明确要求处理缩写，但这个行为与英文实际使用场景存在冲突，应在文档中说明这一局限性或改进切分逻辑。
4. 修复后重新运行 poetry run pytest tests/tokenizer/ -v，需要让所有测试通过。
