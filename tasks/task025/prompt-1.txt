在当前 Python 项目中实现向量时钟域，所有相关代码放在 src/solocoder_py/vector_clock/ 包下，使用内存数据结构记录事件因果关系。
1. 实现向量时钟基础模型：每个节点的时钟是一个从节点 ID 到计数值的映射。节点本地每发生一个事件，对应自己的计数值递增。
2. 实现事件因果偏序判定：给定两个向量时钟，能够判定它们之间是"前者发生在后者之前"、"后者发生在前者之前"还是"两者并发"的关系。
3. 实现并发关系识别：当两个向量时钟互不为对方的前驱时，判定为并发关系。并发关系的判定需要严格按向量时钟的偏序定义来比较。
4. 实现时钟合并：两个向量时钟合并时，每个节点 ID 取两者对应计数值的最大值，产生一个新的向量时钟。
5. 支持时钟查询与比较：可以查询指定节点的当前计数值、比较两个时钟的先后关系（发生在之前 / 发生在之后 / 并发），以及判断两个时钟是否相等。
6. 为该模块编写完整的单元测试，覆盖正常流程（本地事件递增、时钟合并、先后关系判定）、边界条件（空时钟判定、不同节点集的时钟比较）和异常分支（相同计数值的比较、部分节点相同部分不同的并发判定）。测试代码放在 tests/vector_clock/ 下。
7. 在该模块包内编写 README.md，说明模块功能、核心类的职责、向量时钟的偏序判定规则以及使用示例。
