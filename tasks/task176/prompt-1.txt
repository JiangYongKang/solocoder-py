在当前 Python 项目中实现地理位置邻近搜索，所有相关代码放在 src/solocoder_py/geosearch/ 包下，使用内存数据结构存储候选位置点。
1. 实现基于经纬度包围盒的预过滤：给定一个中心坐标和搜索半径（公里），首先构建一个经纬度包围盒（最小/最大纬度和最小/最大经度）。利用纬度一度的弧长近似固定（约 111.32 公里）、经度一度的弧长随纬度变化（约 111.32 × cos(纬度) 公里）的特点，快速计算出给定半径对应的纬度和经度偏移量。对候选点列表做初步过滤，只保留落在包围盒内的点，过滤掉明显超出范围的候选点以减少后续精确计算的开销。
2. 实现 Haversine 精确距离计算与排序：对包围盒过滤后的候选点，使用 Haversine 公式计算每个候选点到中心点的大圆距离（公里）。Haversine 公式需要正确处理球面几何。按计算出的距离升序排序候选点，返回排序后的结果列表，每个结果包含候选点的经纬度以及到中心的精确距离。
3. 支持限制返回数量：搜索结果可以指定返回的最近 N 个点。如果过滤后的候选点数量多于 N，只返回距离最近的 N 个。如果候选点数量不足 N，返回实际找到的全部点。
4. 为该模块编写完整的单元测试，覆盖正常流程（候选点均匀分布在中心点周围时的包围盒过滤正确、Haversine 距离排序与预期远近顺序一致、指定返回数量时的截断正确、包围盒四边附近的候选点被正确保留）、边界条件（空候选点列表搜索、候选点恰好位于包围盒边界线上、搜索半径为 0 时仅返回与中心重合的点、搜索半径极大时包围盒覆盖全球范围、只有赤道附近和高纬度区域的经度偏移差异、跨反子午线的包围盒构建）和异常分支（非法经纬度坐标的校验拒绝、负搜索半径的处理、候选点列表包含非法坐标时的隔离跳过）。测试代码放在 tests/geosearch/ 下。
