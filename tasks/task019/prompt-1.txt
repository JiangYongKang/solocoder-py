在当前 Python 项目中实现幂等键存储域，所有相关代码放在 src/solocoder_py/idempotency/ 包下，使用内存数据结构模拟幂等记录。
1. 实现幂等键生命周期：幂等记录应包含 key、请求指纹、处理状态、响应结果、错误信息、创建时间和过期时间。状态至少包含处理中、成功、失败、已过期。
2. 实现请求指纹绑定：同一个幂等 key 第一次使用时绑定请求指纹；后续使用相同 key 但不同指纹的请求必须被拒绝，避免错误复用结果。
3. 实现并发首请求胜出：多个线程同时使用同一个幂等 key 时，只允许一个请求进入处理中状态，其余相同指纹请求等待或读取最终结果。
4. 实现结果回放：已成功的幂等记录再次被相同请求访问时，应返回原始响应结果，不重复执行业务处理。失败结果是否允许回放需要有明确策略并能配置。
5. 实现 TTL 过期清理：幂等记录超过 TTL 后应在下一次访问时惰性过期，过期后同 key 可以作为新请求重新创建记录。
6. 为该模块编写完整的单元测试，覆盖正常流程（首次处理成功、相同请求结果回放、过期后重新处理）、边界条件（TTL 刚好到期、处理中状态查询、失败结果策略）和异常分支（同 key 不同指纹拒绝、并发首请求胜出、处理失败状态记录）。测试代码放在 tests/idempotency/ 下。
7. 在该模块包内编写 README.md，说明模块功能、核心类的职责、幂等键生命周期与并发处理流程。
