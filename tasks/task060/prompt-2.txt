你实现的分区有序消费模块存在一些问题，需要在 "src/solocoder_py/partition_ordering/" 范围内修复：
1. 重平衡后分区迁移到新消费者时，已提交位点没有被自动继承，可能从旧位置重复消费已提交消息，破坏分区内顺序和位点连续性。
2. 重复分配分区的异常分支实际不可达，所有权冲突可能被静默吞掉，重平衡期间无法可靠发现分区被重复持有。
3. 现有测试通过手动 "seek" 掩盖了重平衡后不会自动恢复位点的问题，还有直接 "pass" 的空测试，不能证明核心需求成立。
4. 补充测试覆盖分区迁移后不手动 seek 也能从已提交位点继续消费、重复分配会产生明确异常、以及强制重平衡期间的冲突处理。
5. 更新 README 中消费者组位点、重平衡和异常处理说明，并运行 "poetry run pytest tests/partition_ordering/ -q"。
