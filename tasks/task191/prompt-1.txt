在当前 Python 项目中实现设备命令队列，使用内存数据结构存储命令和投递状态。
1. 实现 FIFO 保序队列：命令按提交顺序入队，出队时严格保持先进先出顺序。支持基本的入队（enqueue）和出队（dequeue）操作，出队时返回队首命令并将其从队列移除。队列在空时出队应返回 None 或明确标识。
2. 实现投递状态追踪：每条命令在队列中维护其投递状态，包括三种状态：已发送（SENT，命令已从队列取出并投递给设备）、已送达（DELIVERED，设备确认收到命令）、已超时（TIMEOUT，命令发送后在指定时间内未收到设备确认）。提供查询单条命令当前状态和批量查询指定状态命令列表的接口。
3. 实现 TTL 过期：每条命令在入队时可附带一个 TTL（生存时间，单位为秒）。当命令在队列中等待的时间超过 TTL 时，该命令自动变为已超时状态，后续出队时直接跳过并标记为已超时。过期检查可在出队时惰性触发。
4. 代码实现需要放在 src/solocoder_py/cmd_queue/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（命令按 FIFO 顺序出队、状态从 SENT 到 DELIVERED 的转换、批量查询指定状态的命令、TTL 未过期时正常出队）、边界条件（空队列出队返回空、TTL 设为零的命令立即过期、所有命令均已超时时队列行为、大量命令入队出队的顺序正确性、重复确认已送达命令的幂等处理）和异常分支（为不存在的命令 ID 查询状态、确认不存在的命令为已送达、设置负 TTL 的入队拒绝、队列中命令的 ID 重复检测）。测试代码放在 tests/cmd_queue/ 下。
