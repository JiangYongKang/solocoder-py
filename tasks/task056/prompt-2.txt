你实现的配额管理模块存在一些问题，需要在 "src/solocoder_py/quota/" 范围内修复：
1. 当前只支持手动重置配额，没有实现按周期自动重新计量的能力，无法满足按日、按月等周期配额场景。
2. 申请、释放和全局重置之间存在相反的加锁顺序，并发执行时可能互相等待，影响用量统计一致性。
3. 调整租户或全局配额上限时会直接截断已用量，导致租户已用量和全局已用量不一致，后续释放资源可能失败。
4. 补充测试覆盖周期到期重置、全局重置与申请并发交错、以及修改限额后两级用量仍保持一致的场景。
5. 更新 README 中周期重置、并发保证和限额调整语义说明，并运行 "poetry run pytest tests/quota/ -q"。
