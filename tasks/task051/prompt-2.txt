你实现的负载均衡器模块存在一些问题，需要在 "src/solocoder_py/load_balancer/" 范围内修复：
1. 半开熔断实例的探测次数限制没有被严格执行，超过 "half_open_max_probes" 后仍可能被普通可用实例池再次选中，导致熔断恢复阶段放入过多流量。
2. 加权随机策略在直接调用时可能选中权重为 0 的实例，策略自身需要保证零权重实例不会被分配。
3. 补充对应单元测试，覆盖半开探测次数达到上限后的行为，以及直接调用加权随机策略时零权重实例不会被选中的边界场景。
4. 更新 README 中关于熔断半开探测和零权重实例处理的说明。
5. 修复后运行 "poetry run pytest tests/load_balancer/ -q"，确保该模块测试通过。
