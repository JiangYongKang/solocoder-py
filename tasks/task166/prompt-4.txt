当前 metrics 指标度量模块的 quantile 方法修复仍存在缺陷需要处理：
1. quantile 方法在修正 +Inf 桶逻辑后引入了分位数单调性破坏的问题。当所有样本值相同且全部落入 +Inf 桶时（如桶边界为 [1,2,3]，所有样本值均为 100），quantile(0) 通过 _min 路径返回 100，而 quantile(0.5) 在 +Inf 桶插值分支中将下界设为最后一个有限桶边界 3 进行计算，返回 51.5，导致 quantile(0)=100 > quantile(0.5)=51.5，分位数函数不再单调递增。需要修正 +Inf 桶插值下界的取值逻辑，确保分位数在任意分布下保持单调性。
2. 提示词上一轮明确给出了反例场景（桶 [1,2,3]，所有值 100，quantile(0.5) 应接近 100），但代码修复后没有为该反例编写回归测试。需要补充覆盖全部样本落入 +Inf 桶场景的测试用例，确保修复可被测试持续验证。
3. README.md 中的 Histogram 属性文档没有说明新增的 _min 和 _max 字段的含义和用途，开发者无法从文档中了解这两个跟踪字段的存在。
4. 重新运行 `poetry run pytest tests/metrics/ -q`，需要让所有测试通过。
