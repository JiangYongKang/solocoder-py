当前 metrics 指标度量模块存在一些问题需要修复：
1. `quantile` 方法在处理所有样本全部落在 +Inf 桶的场景时存在逻辑缺陷：此时 `lower != upper` 始终为假，方法直接返回最后一个桶边界而非样本的真实观测值。例如桶边界为 [1, 2, 3]，所有样本值均为 100，调用 quantile(0.5) 返回 3 而非 100，调用方会误以为大部分请求延迟在 3 以内。需要修正 +Inf 桶的分位数计算逻辑，使其返回接近真实观测值的结果。
2. `Labels` 类已从公开 API 中移除但没有任何文档或变更日志说明，下游代码通过 `from solocoder_py.metrics import Labels` 导入会直接失败。需要在 README 中记录此 API 变更，说明移除原因和替代方案。
3. 重新运行 `poetry run pytest tests/metrics/ -q`，需要让所有测试通过。
