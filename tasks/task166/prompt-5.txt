当前 metrics 指标度量模块的 quantile 方法仍然存在分位数单调性问题需要修复：
1. 当样本值跨有限桶分布且存在小于第一个桶边界的样本时（如桶边界为 [1,2,3]，样本包含 0.5），quantile(0) 通过 _min 快捷路径返回 0.5，而 quantile(0.0001) 等极接近 0 的分位数在第一桶插值分支中将下界固定设为 0.0 进行计算，导致 quantile(0)=0.5 > quantile(0.0001)=0.0005，分位数函数在 q 接近 0 处不单调递增。现有测试 test_mixed_samples_monotonicity 以 0.1 为步长检查单调性，粒度太粗未能捕获该断层。需要修正第一桶插值下界的取值逻辑，并补充步长足够细（如 0.001 或更小）的单调性验证测试，确保分位数在任意分布和任意查询精度下都保持单调。
2. 重新运行 `poetry run pytest tests/metrics/ -q`，需要让所有测试通过。
