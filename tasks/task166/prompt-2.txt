当前 alert_rule 告警规则评估器的实现存在一些问题需要修复：
1. 数据校验时使用的 InvalidRuleError 异常类未在包的公开 API 中导出，外部无法按具体异常类型捕获和分类处理这类错误。需要将其加入导出列表。
2. 查询冷却期规则列表的方法对"被静默"的判断仅基于冷却时间窗口而不区分当前条件是否真正满足，导致条件本就不满足的规则也被当作因冷却抑制而未触发的情况返回。需要修正查询语义，使其仅返回条件满足但因冷却期而被抑制的那些规则。
3. 批量评估规则时如果某一条规则评估过程抛出异常，整个循环会中断，导致后续规则的结果全部丢失。一条配置错误的规则不应阻止其他规则的正常评估。需要增加对单条规则的异常容错处理。
4. 重新运行 `poetry run pytest tests/alert_rule/ -q`，需要让所有测试通过。
