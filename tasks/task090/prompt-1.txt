在当前 Python 项目中实现多级缓存域，所有相关代码放在 src/solocoder_py/multilevel_cache/ 包下，使用内存数据结构模拟 L1 和 L2 两级缓存。
1. 实现 L1/L2 两级缓存读穿透回填：读取操作首先查询 L1 缓存，如果命中直接返回；如果 L1 未命中但 L2 命中，将数据回填到 L1 后再返回；如果两级都未命中，从可注入的数据源加载数据并依次回填到 L2 和 L1 后再返回。
2. 实现写失效传播：当写入或更新数据时，L1 和 L2 中对应的缓存条目需要同步失效（删除或更新为最新值），保证后续读取能获取到最新数据，不会出现脏读。
3. 实现各级容量独立淘汰：L1 和 L2 各自有独立的容量限制和淘汰策略。L1 容量较小使用 LRU 淘汰，L2 容量较大也可使用 LRU 淘汰。各级缓存的淘汰仅在自身容量超限时触发，L2 淘汰不影响 L1，L1 淘汰也不影响 L2。
4. 为该模块编写完整的单元测试，覆盖正常流程（L1 命中直接返回、L2 命中回填 L1 后返回、两级均未命中穿透加载并回填）、边界条件（L1 满时淘汰触发并正常回填、L2 满时淘汰触发、更新写入后读不再返回旧值）和异常分支（数据源加载失败时两级都不写入、同一 key 被反复淘汰和回填的一致性）。测试代码放在 tests/multilevel_cache/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、读穿透与写失效传播的数据流以及使用示例。
