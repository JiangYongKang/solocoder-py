你实现的 ETL 编排器模块存在一些问题，需要修复：
1. _run_extract 方法在抽取迭代器抛出非致命异常时，将之前所有已成功抽取的数据行全部标记为 ErrorRecord，错误信息均为抽取器失败原因的重复文本。这些行本身已被正确提取，将它们标记为错误会误导调用方以为每行都有数据质量问题，实际问题是数据源迭代器层面的。只需将迭代异常根因记录为一次致命错误，保留已抽取的有效行用于后续阶段处理。
2. exceptions.py 中定义了 StageNotReachableError 异常类并对外导出，README 也声称此异常在"请求的阶段因进度原因不可达时抛出"，但整个 pipeline 执行逻辑中不存在任何抛出该异常的代码路径。该异常要么应在合适的恢复场景中实际抛出，要么应被移除避免给调用方错误预期。
3. _run_extract 方法和 _run_transform 方法中使用了 except BaseException 捕获异常，这会把 KeyboardInterrupt 和 SystemExit 等系统级信号也转换为 FatalEtlError 吞没掉，导致用户按 Ctrl+C 无法正常终止进程。应只捕获业务异常，让系统信号正常传播。
4. _run_extract 方法在异常处理分支中将 extract 阶段标记为已完成却又抛出 FatalEtlError，导致 checkpoint 状态不一致：进度文件显示 extract 已完成，但作业实际是终止状态。恢复执行时会跳过 extract 直接进入 transform，而 transform 依赖的 extracted.pkl 行数据与 checkpoint 计数器可能不一致。
5. 修复后需要重新运行 poetry run pytest tests/etl/ -v，需要让所有测试通过。
