在当前 Python 项目中实现 XML 树解析器，使用内存数据结构构建 DOM 树。
1. 实现 XML 文档解析与 DOM 树构建：解析 XML 文本输入，构建一棵包含元素节点、文本节点、属性节点的树形结构。每个元素节点包含标签名、属性字典、子节点列表和父节点引用。解析器需要正确处理自闭合标签（如 </>）、空元素标签（如 <br/> 或 <br> 的简写形式）、以及混合内容（文本与子元素交替出现）。
2. 实现命名空间处理：支持 `xmlns:prefix="URI"` 形式的命名空间声明。解析后，每个元素可以通过其限定名（qualified name，含前缀）和命名空间 URI 进行查找。提供按命名空间 URI 过滤子元素的方法。
3. 实现属性访问：支持通过属性名获取元素属性值，包括无命名空间前缀的属性和带命名空间前缀的属性。访问不存在的属性时应有明确行为（返回 None 或抛出异常均可，需明确说明）。
4. 实现 XPath 子集求值：支持以下 XPath 表达式的求值：`/`（根节点选择）、`//`（后代节点递归选择）、`*`（通配符匹配任意元素名）、`[@attr='value']`（属性条件过滤）、`[N]`（按位置索引选择子节点，1-based）。表达式可以组合使用，如 `//book[@category='fiction']`。
5. 实现 XML 实体解码：解析过程中正确解码预定义的 XML 实体：`&amp;` → `&`，`&lt;` → `<`，`&gt;` → `>`，`&quot;` → `"`，`&apos;` → `'`。对于数字字符引用（`&#NNN;` 十进制和 `&#xHHH;` 十六进制）也需要正确解析为对应的 Unicode 字符。
6. 代码实现需要放在 src/solocoder_py/xml_parser/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（简单 XML 文档的 DOM 树遍历验证、属性读写、命名空间元素的查询、XPath 表达式求值结果正确、实体解码与原始文本一致）、边界条件（空文档解析、单根元素无子节点、深层嵌套的树结构、大量同名元素的 XPath 选择、纯文本文档（无标签）的处理）和异常分支（不匹配的闭合标签报错、未声明命名空间前缀的使用处理、非法 XPath 表达式语法报错、格式不正确的数字字符引用处理、XML 声明处理、含非法字符的输入拒绝）。测试代码放在 tests/xml_parser/ 下。
7. 在该模块包内编写 README.md，说明模块功能、核心类的职责、DOM 树结构、支持的 XPath 子集语法表、命名空间处理规则、实体解码列表以及使用示例。
