在当前 Python 项目中实现带缓存的记忆化装饰器，使用内存数据结构实现缓存存储与淘汰。
1. 实现按参数生成缓存键：装饰器装饰目标函数后，调用时根据函数名和传入的参数组合生成唯一的缓存键。参数处理需考虑：位置参数和关键字参数统一处理，相同参数值无论以哪种方式传入都应命中同一缓存；对不可哈希类型的参数（如列表、字典）应有降级处理或拒绝的机制。不同函数的缓存空间彼此隔离。
2. 实现 TTL 过期：每个缓存条目在写入时记录创建时间。读取缓存时，如果缓存条目自创建起已超过设定的 TTL（秒），则视为过期，执行原函数重新计算并更新缓存。TTL 可在装饰器使用时配置，支持设为 0 表示永不过期。
3. 实现 LRU 容量淘汰：缓存容量有上限（在装饰器使用时配置）。当缓存条目数达到容量上限且需要插入新条目时，淘汰最近最少使用（Least Recently Used）的条目。每次缓存命中都视为一次"使用"，更新该条目的 LRU 位置。容量可配置，默认值应合理。
4. 实现命中率统计：装饰器需记录缓存的总访问次数和命中次数。提供获取命中率（命中次数 / 总访问次数）的方法。支持重置统计数据的接口，重置后将计数归零。
5. 代码实现需要放在 src/solocoder_py/memoize/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（首次调用执行原函数并缓存结果、相同参数第二次调用直接返回缓存并跳过原函数执行、不同参数各自独立缓存、TTL 过期后重新执行原函数并刷新缓存、LRU 淘汰后最久未使用条目被驱逐、统计命中率计算正确、重置统计归零）、边界条件（TTL 为 0 永不过期的行为、缓存容量为 1 时的 LRU 淘汰、参数值相同但顺序不同的位置参数和关键字参数命中同一缓存、返回值为 None 时缓存的正确性、缓存容量远大于实际缓存条目数时的行为）和异常分支（不可哈希类型参数（列表、字典）的处理策略、装饰器应用到非函数对象上的报错、同时设置 TTL 和 LRU 且两者同时触发时的交互行为、装饰器应用到带可变参数（*args, **kwargs）函数上的缓存键生成）。测试代码放在 tests/memoize/ 下。
