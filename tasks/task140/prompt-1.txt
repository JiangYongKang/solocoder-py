在当前项目中实现一个 API 版本内容协商器功能模块，使用内存数据结构模拟数据源。
1. 实现 Accept-Version 头路由分发：请求携带 Accept-Version 头声明期望的 API 版本号（如"v1"、"v2"），内容协商器根据该头将请求路由到对应的版本处理器。如果没有携带 Accept-Version 头，则使用默认版本（通常是当前最新稳定版本）。版本号格式支持语义化格式（如 v1、v2.0、v2024-01）。
2. 实现版本匹配策略：支持精确版本匹配（请求 v1 只能匹配 v1）和兼容版本匹配（请求 v1 也可匹配标注为兼容 v1 的 v2 处理器）。当请求的版本不存在且无可兼容版本时，返回明确的版本不支持错误，同时在响应中列出可用版本列表。
3. 支持废弃版本通知：对标记为已废弃（Deprecated）的版本，在此版本上处理的请求在正常返回响应的同时，额外在响应头中附加废弃通告信息（如 Sunset 头指定废弃生效日期、Deprecation 头标记为 true），提醒客户端尽快迁移到新版本。
4. 代码实现需要放在 src/solocoder_py/version_negotiator/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程、边界条件（如未携带版本头时使用默认版本、请求的版本恰好是废弃版本时的双路径响应）和异常分支（如请求完全不存在的版本且无兼容版本、同一处理器注册了冲突版本号的重复注册防范、废弃版本的 Sunset 日期已过期的请求拒绝行为）。测试代码放在 tests/version_negotiator/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、Accept-Version 头的解析规则、版本匹配与兼容策略、废弃版本的通知机制以及使用示例。
