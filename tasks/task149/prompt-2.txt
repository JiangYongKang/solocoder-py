在当前项目中修复 pathfinding 网格寻路器的一些问题：
1. 当前代码中存在两套功能重叠但接口不兼容的寻路实现：grid.py + astar.py 是被实际引用的主实现，grid_map.py + a_star.py 是一套孤立且存在严重问题的冗余实现。a_star.py 和 grid_map.py 中存在多处导入不存在的异常类和使用不存在的字段名，导入时即会崩溃。这组文件未被 __init__.py 导出、未在任何测试中引用，属于无论如何都无法运行的死代码，需要清理。
2. astar.py 中 A* 算法的 find_path 方法将起始节点的 f_score 硬编码为 0.0，而算法标准做法应使用启发式函数值 h(start, goal) 作为起始节点的 f 值。虽不影响最终结果但引入了与其他节点的算法不一致性，应使用统一的 f = g + h 公式。
3. GridMap 类的 is_passable 方法对越界坐标静默返回 False，而 get_weight 方法对越界坐标则抛出异常。两者同属越界查询操作却采用不同处理策略，接口行为不一致增加了调用方的心智负担，应统一处理方式。
4. conftest.py 中存在多个从未被测试引用的辅助函数，且这些函数使用了当前 API 中不存在的参数名，属于折旧代码需要清理。
5. 修复后重新运行 poetry run pytest tests/pathfinding/ -q，确保所有测试通过。
