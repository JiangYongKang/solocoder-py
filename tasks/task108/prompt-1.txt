在当前 Python 项目中实现 Base64/Base32/Base16 流式编解码器，所有相关代码放在 src/solocoder_py/basecodec/ 包下，使用内存数据结构模拟字节流读写。
1. 实现 Base64、Base32 和 Base16 三种编解码算法：编码器将原始字节流按对应算法转换为文本字符串，解码器将文本字符串还原为原始字节流。Base64 使用 64 个可打印字符（A-Z、a-z、0-9、+、/），Base32 使用 32 个字符（A-Z、2-7），Base16 使用 16 个字符（0-9、A-F）。每种编解码都支持流式处理，即可以分批输入数据、分批输出结果，不需要一次性加载全部数据。
2. 实现无填充模式：标准编解码在输入数据长度不是编码单元整数倍时会在末尾添加填充字符（如 Base64 的等号）。无填充模式下省略填充字符，编码结果的字符总数完全由输入数据长度决定。解码时自动根据编码结果的长度推算原始数据长度，不需要填充字符作为标记。
3. 实现行宽折行控制：Base64 编码常用于邮件等文本协议，需要控制每行最大字符数并在行尾插入换行符。编码时支持指定行宽（如每 76 个字符插入换行），解码时自动过滤输入中的换行字符和其他空白字符。行宽为零表示不折行。
4. 为该模块编写完整的单元测试，覆盖正常流程（Base64 编码与解码一致性、Base32 编码与解码一致性、Base16 编码与解码一致性、分批输入数据的流式编码和解码）、边界条件（空字节流编码为空字符串、单字节输入、填充边界输入长度如恰好 3 字节的 Base64 编码无填充、含换行符的编码结果正确解码、行宽刚好整除编码总字符数）和异常分支（非法字符输入解码时报错、截断的编码结果解码长度不正确、无填充模式下输入长度与编码结果长度不匹配的校验、非法的 Base32/Base16 字符输入处理）。测试代码放在 tests/basecodec/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、三种编码算法的编码表与规则、无填充模式与标准模式的区别、行宽折行控制方式以及使用示例。
