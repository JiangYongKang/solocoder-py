在当前 Python 项目中实现紧凑二进制序列化器，所有相关代码放在 src/solocoder_py/serializer/ 包下，使用内存数据结构模拟读写缓冲区。
1. 实现变长整数编码：整数序列化时不使用固定字节数，而是根据数值大小使用 1 到多个字节。小数值用少量字节编码，大数值用更多字节，每个字节的最高位作为继续标志位指示后续还有无字节。变长整数支持无符号和有符号两种，有符号整数使用 ZigZag 编码将负数映射为正数后再做变长编码。
2. 实现 ZigZag 编码：有符号整数（可为负值）通过 ZigZag 映射转换为无符号整数，映射规则为：0→0、-1→1、1→2、-2→3、2→4……依次交错排列。这样做的好处是绝对值较小的负数编码后占用的字节数也较少，适合实际场景中常见的小负数。
3. 实现 Schema 向前兼容演进：序列化器支持 Schema 版本管理。每个可序列化的数据结构定义自己的 Schema 和版本号。新版本 Schema 可以添加新的字段（只能追加不能删除或修改已有字段的类型），旧版本的数据可以被新版本 Schema 反序列化（新字段使用默认值）。反序列化时遇到未知的字段编号应忽略而不是报错，保证向前兼容。
4. 为该模块编写完整的单元测试，覆盖正常流程（正整数变长编码解码、负整数 ZigZag 编码后变长编码解码、零值编解码、基本类型如布尔值和字符串的序列化与反序列化、Schema 升级后旧版本数据用新 Schema 反序列化正确填充默认值）、边界条件（零值变长编码、最大整数值的变长编码边界、空字符串序列化、Schema 无新增字段时新旧版本数据完全一致、单字节与多字节变长整数的边界值）和异常分支（变长编码字节截断后的解码错误、ZigZag 编码溢出的处理、Schema 版本不兼容（旧 Schema 遇到新字段）时忽略未知字段、删除必需字段的 Schema 兼容性校验失败）。测试代码放在 tests/serializer/ 下。
