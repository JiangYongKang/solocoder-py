2. 模型在 serializer.py 的 _write_field_value 方法中新增了严格类型检查，这个逻辑本身是正确的，但 README.md 中对 BOOL 字段描述的表述存在歧义——写的是"BOOL 字段只接受 bool 类型（传入 int 或 str 都会报错）"，紧接着说 BOOL 也会被整数字段拒绝，容易让读者误以为传入 int 到 BOOL 字段时会触发双重检查。应在 BOOL 字段的描述中只说明它只接受 bool，把 bool 是 int 子类这个上下文放在整数字段的检查逻辑中集中说明。
