在当前 Python 项目中实现登录限流与渐进锁定域，使用内存数据结构管理登录失败计数与锁定状态。
1. 实现按账户与 IP 子网两级计数：对每个账户单独维护登录失败计数器，同时对登录请求的来源 IP 所在 /24 子网（即 IP 前三段相同）也维护一个独立的失败计数器。每次登录失败时同时递增账户计数器和对应子网计数器。成功登录时重置该账户和对应子网的计数器为零。
2. 实现阶梯式等退：登录失败后不是固定等待时间，而是根据累计失败次数逐步延长下一次允许尝试的等待时间。例如第一次失败后等待 1 秒，第二次 2 秒，第三次 4 秒，以此类推成倍增长直至达到预设的上限（最大等退时间）。在等退倒计时内，即使输入正确密码也不进行验证，直接返回"请等待 X 秒后重试"。
3. 实现账户锁定与 CAPTCHA 升级钩子：当账户失败次数达到设定阈值后被锁定，此后所有该账户的登录请求直接返回"账户已锁定"而不进行密码校验。当子网失败次数达到阈值时，该子网下的所有登录请求均被要求附加 CAPTCHA 验证（提供一个钩子接口，外部可以注入 CAPTCHA 验证逻辑），CAPTCHA 验证通过后即使账户在锁定状态也允许进入密码校验。锁定状态可以通过管理员接口手动解锁。
4. 代码实现需要放在 src/solocoder_py/login_rate/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（登录失败计数递增、阶梯等退时间正确、账户锁定阈值触发、子网计数器独立于账户计数器）、边界条件（等退时间执行上限不无限增长、同一子网下不同账户分别计数、成功登录重置计数、锁定后管理员解锁）和异常分支（等退期间无密码校验、锁定后直接拒绝登录、无效账户名被拒绝、子网格式非法被拒绝、不存在的账户计数器访问）。测试代码放在 tests/login_rate/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、阶梯等退算法、账户/IP 双维度限流策略与 CAPTCHA 升级机制（可用文本描述）以及使用示例。
