当前 login_rate 登录限流与渐进锁定的实现存在一些问题需要修复：
1. 阶梯等退期间的错误提示和账户锁定后的错误提示均使用了英文消息，与设计要求的用户可见中文提示文案不一致。需要将所有面向终端用户的错误提示统一为中文。
2. 检查账户是否被锁定的方法与同类查询方法对不存在账户的处理行为不一致——一个返回默认值，另一个抛出异常。调用方无法区分"未锁定"和"账户从未存在过"两种情况。需要保持 API 语义一致。
3. 登录尝试方法中存在一段无效的异常处理代码——捕获异常后不做任何处理就原样重新抛出。需要对所有方法做一轮死代码清理，移除无效的代码路径。
4. README 中的示例代码引用的是英文错误消息文本，与实际代码输出的文案不匹配，用户按文档运行示例代码会断言失败。需要更新文档中的示例以与代码实际行为一致。
5. 重新运行 `poetry run pytest tests/login_rate/ -q`，需要让所有测试通过。
