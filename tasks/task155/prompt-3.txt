你实现的 diff 模块（Myers 文本差异对比模块）存在一些问题，需要修复：
1. 提示词要求三级粒度可以独立选择或组合使用，但当前 diff 模块只支持单一粒度参数。例如先按行做第一轮比较找出变更行，再在变更行内按词做第二轮精确标注的组合场景无法实现。
2. 上下文行的收集逻辑有问题。上下文行数按整个 DiffOperation 包含的 token 数量来判断，而不是按单个 token 逐个计数。这导致当某段相同内容包含的 token 数量超过配置的上下文行数时，整段上下文被直接丢弃，生成的 hunk 缺少应有的上下文行。
3. hunk header 的行号输出存在索引约定不一致。行级粒度输出的起始行号是 1-based，词级和字符级粒度输出的起始行号是 0-based，不同粒度输出的 header 行号含义不统一，外部解析时容易产生 off-by-one 偏移。
4. 修复后重新运行 poetry run pytest tests/diff/ -v，需要让所有测试通过。
