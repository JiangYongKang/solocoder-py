你实现的 Myers 文本差异对比模块存在一些问题，需要修复：
1. 提示词要求"三级粒度可独立选择或组合使用"，但当前只支持单一粒度参数，无法实现例如先按行比较再在变更行内做词级精确标注的组合场景。
2. build_hunks 函数中上下文行的收集以整个 DiffOperation 的 token 数量作为粒度计数而非以单个 token 为单位。当变更前后某个 EQUAL 操作包含的 token 数量超过设定的上下文行数时，该操作对应的全部上下文行被整体丢弃，导致生成的 hunk 缺少应有的上下文。
3. format_hunk_header 方法对不同粒度使用不同的索引约定——行级输出 1-based 起始行号，词级和字符级输出 0-based 起始行号。这种不一致会让外部代码解析不同粒度 diff 输出的 hunk header 时产生 off-by-one 偏移。
4. 修复后重新运行 poetry run pytest tests/diff/ -v，需要让所有测试通过。
