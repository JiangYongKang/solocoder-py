产物不满意：
1. 提示词要求"三级粒度可独立选择或组合使用"，但 TextDiffer.diff 只接受单个 granularity 参数且内部只调一次 tokenize，没法实现比如先按行比较找到变更行、再在变更行内按词标注更细差异的组合场景。根因是只做了独立选择，把"组合使用"完全忽略了。应该支持传入粒度列表，按优先级逐级精化差异结果。
2. build_hunks 里上下文行的收集以整个 DiffOperation 的 token 数做粒度计数，不是以单个 token 为单位。当某个 EQUAL 操作的 token 数超过 context_lines 时，代码直接 break 放弃全部上下文 token。比如 context_lines=3 但有个包含 4 个 EQUAL token 的操作，就会导致整个 hunk 没有任何上下文行。测试只验了上界没验实际上下文数量。应该在循环内对单个操作按需逐 token 引入上下文。
3. format_hunk_header 对行级粒度输出 1-based 起始行号（old_start+1），但对词级和字符级输出 0-based（直接用 old_start）。这种随粒度切索引约定的行为没任何文档说明，外部代码按标准 Unified Diff 的 1-based 约定解析非行级 diff 输出就会产生 off-by-one 偏移。应该所有粒度统一用 1-based 约定。

过程不满意：
1. 模型收到提示词第一步就建了 8 项计划，然后去读 three_way_merge 包的 models.py、lcs.py 等四个文件来了解项目约定。three_way_merge 跟 diff 任务完全无关，通过读无关包来学项目结构是绕路的低效行为。应该先精读提示词明确指定的 diff 包路径和 pytest 约定，查看 pyproject.toml 了解构建和测试配置，再根据 diff 域的实际需求规划。
2. 第一次跑 pytest 发现 Myers 算法回溯有问题后，模型没先看 traceback 定位出错行，直接用 Write 工具把 myers.py 整个文件重写了。然后又创建临时脚本单独验证，而不是先用标准命令验证是否覆盖了所有失败用例。这是试错式调试，先整体重写再单独验证，而不是基于失败信息做最小范围修正。
3. 模型全程用 python -m pytest 跑测试而不是项目约定的 poetry run pytest，虽然功能等价但违反了构建规范。poetry 管理的虚拟环境靠 poetry run 保证依赖和路径映射一致，直接裸 python 调用可能出包导入路径问题。
