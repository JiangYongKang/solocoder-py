在当前 Python 项目中实现颜色空间转换器，使用纯数值计算实现各颜色空间之间的互转。
1. 实现颜色空间互转：支持 RGB（0-255 整数或 0-1 浮点）、HSL（色相 0-360、饱和度 0-100%、亮度 0-100%）、HSV（色相 0-360、饱和度 0-100%、明度 0-100%）、HEX（#RRGGBB 或 #RGB 简写格式）四种颜色表示之间的相互转换。每种颜色提供对应的数据类或命名字段表示，便于从不同空间转换到目标空间。
2. 实现色域裁剪：当转换过程中某个通道的值超出目标颜色空间的合法范围时（例如 RGB 转 HSL 后再转回 RGB 因浮点精度产生超出 0-255 的值），自动将超出范围的值裁剪到最近的边界值（clamping），而不是抛出异常或返回无效颜色。
3. 实现 Alpha 合成：提供将两个 RGB 颜色按 Alpha 透明度进行合成的功能（前景色叠加到背景色上），按照 Alpha 合成公式（over 操作）计算合成后的 RGB 值。Alpha 值范围为 0-1（0 完全透明，1 完全不透明）。支持两个颜色都带 Alpha 的合成。
4. 实现 WCAG 对比度计算：给定两个颜色，计算它们的相对亮度（relative luminance），然后根据 WCAG 2.0 对比度公式计算对比度比值。提供一个便捷方法判断两个颜色的对比度是否满足 WCAG AA（4.5:1）和 AAA（7:1）级别的正常文本标准。
5. 代码实现需要放在 src/solocoder_py/colorspace/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（RGB↔HEX 互转的精准性、RGB↔HSL 互转后的往返精度、RGB↔HSV 互转后的往返精度、HSL↔HSV 互转的正确性、Alpha 合成的标准场景（半透明白色叠加到黑色）、#RGB 简写的展开、WCAG 对比度标准场景（黑底白字为 21:1 最高对比度、白底白字为 1:1 最低对比度）、WCAG 级别判定的正确性）、边界条件（纯黑纯白的各空间表示、RGB 三个通道均为零、RGB 三个通道均为 255、饱和度为 0 的灰色调 HSL 色相值的处理、HEX 格式大小写不敏感的解析）、异常分支（非法 HEX 字符串（长度不为 3/4/6/8、含非法字符）的解析拒绝、RGB 各通道超出 0-255 时的裁剪、HSL/HSV 色相超出 0-360 时的归一化、饱和度和亮度超出 0-100% 时的裁剪、Alpha 值超出 0-1 范围的裁剪）。测试代码放在 tests/colorspace/ 下。
