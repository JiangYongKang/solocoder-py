你实现的按标签缓存失效模块存在一些问题，需要修复：
1. _purge_expired 方法每次只扫描存储字典的前 100 个条目，遇到非过期条目就退出，导致位置 100 之后的所有过期条目永远不会被扫描清理。这会造成过期条目长期占用内存，entry_count 计数虚高。需要确保所有过期条目都能被覆盖扫描到。
2. exceptions.py 中定义了 TagNotFoundError 异常类并被导出，但所有标签查询方法都选择了静默处理不存在的标签（返回空列表或 False），没有一处抛出该异常，形成了死代码。调用方会预期不存在的标签触发异常，但实际行为是静默返回，API 契约不一致。需要删除死代码异常或统一标签查询的异常抛出行为。
3. remove_tags 方法在遍历标签列表时缺少对 tag is None 的校验，而 add_tags、set、invalidate_tag 等其他标签操作都有此校验。调用方传入包含 None 的标签列表时，remove_tags 静默处理，导致 API 行为不一致。
4. 修复完成后重新运行 poetry run pytest tests/tag_cache/ -v，确保所有测试通过。
5. 更新 README.md，说明过期清理策略的扫描覆盖机制以及标签校验的一致性约定。
