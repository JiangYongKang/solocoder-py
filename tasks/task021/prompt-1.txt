在当前 Python 项目中实现消费侧收件箱去重域，所有相关代码放在 src/solocoder_py/inbox/ 包下，使用内存数据结构模拟消息接收记录。
1. 实现消息 ID 滑动窗口去重：根据消息唯一 ID 判断是否重复，去重窗口按已接收消息数量或时间跨度滑动，窗口外的旧 ID 不再参与去重比对。
2. 实现乱序到达处理：消息到达顺序可能与发送顺序不一致，在窗口范围内即使后发送的消息先到达，也应能正确去重，不依赖消息到达顺序。
3. 实现过期清理：去重记录超过 TTL 后应被清理，清理后的 ID 再次到达时视为新消息。支持惰性过期和主动定时清理两种方式。
4. 支持去重窗口配置：窗口大小（最大记录数或最大时间跨度）和 TTL 均可配置，并支持查询当前窗口内的去重记录数量和去重命中率。
5. 保证并发安全：多个线程同时接收和处理消息时，同一条消息不能被判断为非重复多次消费。
6. 为该模块编写完整的单元测试，覆盖正常流程（消息去重、窗口滑动、过期清理后重新接收）、边界条件（窗口刚好满、刚好到 TTL、空窗口）和异常分支（并发去重不重复消费、同一消息在窗口边界外再次到达不判定重复）。测试代码放在 tests/inbox/ 下。
7. 在该模块包内编写 README.md，说明模块功能、核心类的职责、滑动窗口去重原理以及使用示例。
