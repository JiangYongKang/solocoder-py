在当前 Python 项目中实现 ECS 实体组件系统域，使用内存数据结构模拟游戏世界。
1. 实现实体与组件的创建和关联：实体是薄 ID 标识，不承载任何数据或行为。组件是纯数据结构，存储实体的属性数据（如位置、速度、生命值等）。一个实体可以关联零到多个不同类型的组件。支持按组件类型查询所有拥有该组件的实体集合。
2. 实现稀疏组件存储：组件数据使用稀疏集合存储，使得按实体 ID 访问组件有常数时间复杂度，按组件类型遍历所有实例有紧凑的内存连续性和缓存友好性。组件的增删操作应高效。
3. 实现系统按依赖拓扑排序调度：系统（System）是对拥有特定组件集合的实体进行批量处理的逻辑单元。每个系统声明其读写依赖的组件类型，根据组件类型的读写依赖关系自动推导系统间的执行顺序。存在循环依赖时应在初始化阶段检测并报错。
4. 实现原型（Archetype）分组优化：将拥有完全相同组件类型组合的实体归入同一原型，同一原型内的所有实体数据按列（SoA，数组结构体）连续存储。遍历拥有特定组件组合的实体时直接在匹配的原型中批量迭代，避免逐实体做组件存在性检查。
5. 代码实现需要放在 src/solocoder_py/ecs/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（实体创建与组件关联、组件类型查询、系统按拓扑顺序执行）、边界条件（系统无依赖时的执行、单一原型的实体遍历）、异常分支（循环依赖检测并报错、移除组件后原型重新分组、对不存在组件的实体执行系统不报错）。测试代码放在 tests/ecs/ 下。
6. 在该模块包内编写 README.md，说明模块功能、核心类的职责、原型分组与系统调度机制（可用文本描述）以及使用示例。
