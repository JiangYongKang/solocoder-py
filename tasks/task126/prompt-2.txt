你实现的自动补全索引存在一些缺陷，需要修复：
1. 大小写不敏感模式下，insert、contains、get_weight、update_weight 和 delete 方法使用原始单词作为 _word_weights 字典的键而非归一化后的键。导致 "Apple" 和 "apple" 被当作两个独立词条分别插入，update_weight("test") 在已存在 "Test" 时不会更新已有词条而是新建条目，delete("hello") 在已插入 "Hello" 后返回 False 无法删除。需要在 _word_weights 的读写操作前统一做大小写归一化处理，保证字典键与 Trie 路径的大小写行为一致。
2. 提示词要求支持"更新候选项的热度或文本"，但当前只实现了 update_weight 用于更新热度，没有任何方法支持修改候选项的文本内容。如果要改文本，只能先 delete 再 insert，但这样会丢失原有热度值。需要增加更新候选项文本的功能，保留热度的同时修改词条文本。
3. 修复后需要为大小写归一化键一致性和文本更新功能补充单元测试。测试代码放在 tests/autocomplete/ 下。
4. 更新 README.md，说明大小写不敏感模式下的内部键归一化策略和文本更新功能的使用方式。
5. 重新运行 poetry run pytest tests/autocomplete/ -q，确保所有测试通过。
