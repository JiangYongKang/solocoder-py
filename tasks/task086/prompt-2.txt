你实现的优先级线程池模块存在一些缺陷，需要修复：
1. threadpool.py 的 shutdown 方法和 run_until_complete 方法中，通过 hasattr(self._clock, "advance") 判断时钟类型来决定如何推进时间。当使用 SystemClock（没有 advance 方法）且任务 duration 大于零时，循环内遇到 running 任务后直接 break 跳出，导致任务被丢弃从未完成。需要为 SystemClock 路径提供等效的时间等待机制，如使用条件变量或事件等待。
2. submit_many 方法无法传递 duration 参数。该方法以 (func, args, kwargs, priority) 元组列表作为输入，内部通过 submit 调用，但 kwargs 字典不含 duration 字段，方法签名也未单独接收 durations 列表。需要修改 submit_many 增加对 duration 参数的支持，或者在调用 submit 时显式抽取 duration。
3. README.md 对 get_task_result 方法的描述有误，写的是"任务未完成时抛出 TaskNotFoundError"，但实际代码对尚未完成的任务返回包含当前状态的 TaskResult 对象，不会抛异常。只有 task_id 完全未注册时才会抛出 TaskNotFoundError。需要修正文档描述。
4. models.py 的 TaskStatus 枚举定义了 CANCELLED 和 REJECTED 两个状态值，但代码中从未将任何任务设置为这两种状态。关闭线程池时拒绝新任务直接抛异常而非创建 REJECTED 状态的 TaskResult，CANCELLED 对应的取消功能完全没有实现。需要删除未使用的枚举值或在相应场景中实际使用它们。
5. get_task_result 方法对尚未开始的任务返回 TaskResult 时，started_at 和 completed_at 字段默认值为 0.0，与 submitted_at 的有效时间戳产生语义不一致。调用者无法区分"任务在 t=0 时开始执行"还是"任务尚未开始"。应使用 None 表示"尚未开始"和"尚未完成"，或增加执行状态字段做明确区分。
6. 修复完成后重新运行 poetry run pytest tests/priority_threadpool/ -q，确保所有测试通过，并为 SystemClock 路径补充至少一个测试用例。
