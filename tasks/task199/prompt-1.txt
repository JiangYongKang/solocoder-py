在当前 Python 项目中实现依赖注入容器，使用内存数据结构管理类型的注册与实例化。
1. 实现三种生命周期：Singleton（单例）：容器中该类型只创建一个实例，所有请求共享同一个实例。Transient（瞬时）：每次请求该类型时都创建一个新实例。Scoped（作用域）：在同一个作用域内只创建一个实例，不同作用域之间实例隔离。作用域可以从容器创建，作用域结束后作用域内的 Scoped 实例不再可用。
2. 实现构造函数注入：容器根据类型的构造函数参数自动解析依赖。注册类型到容器时，容器分析类型的 __init__ 方法签名，当请求该类型实例时自动递归解析并注入构造函数所需的所有参数。构造函数参数可以是注册的基础类型、接口类型或具体实现类型。
3. 实现循环依赖检测：在解析类型的依赖链时，如果发现 A 依赖 B、B 依赖 A 等循环依赖情况，应检测到循环并抛出明确的异常（其中包含循环链中涉及的类型名称），而不是陷入无限递归导致栈溢出。
4. 实现类型注册接口：提供向容器注册类型的方法。支持按类型注册（key 为类型本身），也支持按接口注册（key 为抽象类或接口，注册具体实现）。请求实例时可指定待解析的类型名称。支持注册时指定生命周期。
5. 代码实现需要放在 src/solocoder_py/di/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（Singleton 类型多次请求返回同一实例、Transient 类型每次请求返回新实例、Scoped 类型同作用域内返回相同实例不同作用域返回不同实例、构造函数自动注入正确参数、多层嵌套依赖的递归注入、按接口注册和解析）、边界条件（无构造函数参数的类型正常解析、Singleton 生命周期下构造函数仅执行一次、嵌套 Scoped 作用域的隔离性验证、空容器的请求报错）和异常分支（检测直接循环依赖（A → B → A）并抛出描述性异常、检测间接循环依赖（A → B → C → A）并抛出异常、请求未注册类型的明确报错、构造函数参数无法解析时的精准报错、Scoped 实例在作用域结束后再请求的处理、注册时生命周期参数非法值的拒绝、单例类型的类型重复注册冲突处理）。测试代码放在 tests/di/ 下。
6. 在该模块包内编写 README.md，说明模块功能、核心类的职责、三种生命周期的语义与使用场景、构造函数注入的自动解析策略、循环依赖检测算法、类型注册接口与配置方式以及使用示例。
