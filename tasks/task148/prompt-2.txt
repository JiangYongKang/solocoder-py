在当前项目中修复 collision 碰撞检测引擎的一些问题：
1. Collider 在 __post_init__ 中校验 collider ID 为空时抛出了 InvalidAABBError 异常，但空 ID 属于 collider 元数据校验范畴而非 AABB 几何范围校验，异常类型的语义与实际错误不匹配。调用方排查时会先查包围盒坐标而非 collider 身份标识，增加调试成本。应让 AABB 类只校验坐标合法性，Collider 独有的身份校验使用语义匹配的异常类型。
2. spatial_hash.py 的 get_collider 方法体内使用了内联导入语句导入 ColliderNotFoundError，而文件顶部已有其他同级导入。这种将导入写在方法内部的做法违反编码规范，其他开发者浏览文件顶部时无法一眼看清完整依赖。应将此导入移至文件顶部与其他导入合并。
3. CollisionPair 的 __post_init__ 在初始化时静默按 ID 字典序交换了传入的两个 collider 字段值以实现规范化排序。调用方传入的第一个 collider 在构造完成后可能不再是原先传入的那个对象，这种静默修改 dataclass 属性的设计容易导致调用方通过字段名访问时产生困惑。应在文档中明确说明或在 API 层面减少惊讶。
4. 修复后重新运行 poetry run pytest tests/collision/ -q，确保所有测试通过。
