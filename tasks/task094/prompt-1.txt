在当前 Python 项目中实现 Schema 版本迁移运行器，使用内存数据结构模拟 Schema 版本与迁移脚本。
1. 实现顺序升级：每个迁移脚本对应一个目标版本号，迁移运行器按照版本号从小到大的顺序依次执行尚未应用的迁移脚本。例如当前 Schema 版本为 2，存在目标版本为 3、4、5 的迁移脚本，则应依次执行版本 3、4、5 的脚本，最终将 Schema 升级到版本 5。不允许跳过中间版本直接执行更高版本。
2. 实现幂等执行：每个迁移脚本必须保证幂等性——同一个迁移脚本即使被重复执行多次，Schema 的最终状态与执行一次完全相同。迁移运行器在提交迁移结果时应校验脚本的幂等性，如果重复执行导致数据不一致应标记为迁移脚本编写问题。
3. 实现失败时按相反顺序回滚：如果某个迁移脚本执行失败，已成功执行的前序迁移脚本应按从高版本到低版本的相反顺序依次执行回滚，将 Schema 恢复到迁移开始前的状态。例如版本 3 和 4 执行成功但版本 5 失败，则应执行版本 4 的回滚再执行版本 3 的回滚。
4. 代码实现需要放在 src/solocoder_py/schema_migration/ 包下，然后为该模块编写完整的单元测试，覆盖正常流程（多个版本顺序升级全部成功、从零版本初始化到最新版本）、边界条件（只有一个待执行迁移脚本、当前已是最新版本无需迁移、迁移脚本列表为空）和异常分支（中间版本执行失败后的反向回滚、单个迁移脚本重复执行验证幂等性、回滚过程中某个回滚脚本也失败的容错处理）。测试代码放在 tests/schema_migration/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、顺序升级与回滚的执行规则以及使用示例。
