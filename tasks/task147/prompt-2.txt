在当前项目中修复 game_loop 固定时间步游戏循环的一些问题：
1. tick 方法中的插值逻辑实现的是前向外推（将当前状态拷贝为起点再对其做一次 update 预测下一状态后在两者间插值），而非提示词要求的基于"最近两次逻辑帧实际状态的插值"。当前实现中渲染位置比实际逻辑状态领先一个步长，当游戏对象改变速度或方向时会导致视觉过冲和状态回弹，不符合平滑无抖动的设计要求。需要改为保存上一逻辑帧的真实状态，在当前帧和上一帧之间做 alpha 插值。
2. 跳帧阈值的实现与 README 文档描述不一致：README 写的是超过 max_catch_up_steps × time_step 即跳帧，但代码中实际使用了 10 倍的额外乘数且无文档说明。需要统一实现与文档，给出了具体乘数则需要明确说明设计理由。
3. tick 方法热路径上每帧都执行一次插值能力试探调用（调用 interpolate 然后丢弃结果），这种做法伴随着不必要的深拷贝和 update 开销。应在初始化阶段确定状态是否支持插值，而非每帧都做代价高昂的试探。
4. InterpolatedState 的 previous_state 字段在 tick 的所有返回路径中都被设为 None，get_interpolated 中针对该字段的回退逻辑从未被触发。应将这个未使用的字段从 API 中移除或在 tick 中正确维护它。
5. 修复后重新运行 poetry run pytest tests/game_loop/ -q，确保所有测试通过。注意修复插值逻辑后对应的测试期望值也需要更新为正确的插值结果。
