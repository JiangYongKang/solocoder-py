leader_election 包存在几个比较严重的缺陷，在当前包内修复。
1. run_election 不检查也不清理旧领导者。集群已有 node-0 作为领导者时，直接让 node-1 发起选举并当选，node-0 仍保持在 LEADER 状态不变，集群同时出现两个领导节点，违反了脑裂防护的基本要求。新领导当选后必须让旧领导退位。
2. 完全没有选举超时机制。提示词要求节点在超时未收到领导心跳时自动发起选举，但当前代码里节点从不主动检查心跳间隔并触发选举，所有选举都靠外部手动调用。需要实现心跳监控和选举超时逻辑，让节点能够在领导者失联一段时间后自动发起选举。
3. leader_send_heartbeat 用裸 except Exception 吞掉了 StaleTermError。跟随者因分区期间任期已升高而拒绝旧领导心跳时，异常被静默忽略，领导者感知不到自己已过期、不会主动退位。心跳失败时应该识别任期落后并自动退位。
4. handle_vote_request 对同任期同候选人的重复投票请求仍返回 vote_granted=True，如果被重复调用会导致得票计数被重复追加，超出实际票数。同一个任期对同一个候选人只能投一次票。
5. 修完跑 poetry run pytest tests/leader_election/ -q，确保全过。
6. README 同步更新，说明选举超时机制、脑裂防护策略和心跳异常处理规则。
