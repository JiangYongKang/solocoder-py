当前 session_store 会话存储的实现存在一些问题需要修复：
1. 被淘汰和被强制下线的会话对象在会话 ID 索引中始终残留，没有任何清理机制，长期运行的服务会因此产生内存泄漏。需要确保这些已被下线、不再需要完整服务的会话对象能从所有索引结构中正确清理。
2. 会话数据类中对不同字段的校验使用了不一致的异常类型——部分字段抛 Python 内置异常，部分字段抛领域自定义异常。外部调用方无法用统一的领域基异常来捕获校验错误。需要统一异常层级，使数据模型的字段校验错误落在领域异常体系内。
3. 同一套配置校验规则（TTL 必须大于零、空闲超时必须小于等于 TTL 等）在当前代码库的三个不同位置被重复维护，错误消息逐字相同，未来变更需要三处同步修改。需要将校验规则集中到唯一的权威来源处。
4. 重新运行 `poetry run pytest tests/session_store/ -q`，需要让所有测试通过。
