当前 metrics 指标采集器的实现存在一些问题需要修复：
1. README 中对 Histogram 桶区间的语义描述与实际代码行为不一致，开发者按文档理解桶边界值的归属时会产生错误预期。需要统一文档描述和代码实现中桶区间的开闭语义，消除矛盾。
2. Labels 类在公开 API 中导出但在所有生产代码中从未被使用，成为了冗余的公开 API。需要将此类的职责整合进实际生产流程中，或从其导出列表中移除，避免给用户暴露无实际功能的类。
3. Histogram 分位数计算方法未覆盖最后一个桶的数据，导致大量样本落入最后一个桶时分位值被严重低估。需要修正分位数计算的循环范围，令其对全部桶元素进行正确的累计统计。
4. 重新运行 `poetry run pytest tests/metrics/ -q`，需要让所有测试通过。
