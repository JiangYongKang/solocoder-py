在当前 Python 项目中实现单飞（single-flight）请求合并器域，所有相关代码放在 src/solocoder_py/singleflight/ 包下，使用内存数据结构协调并发请求。
1. 实现同 key 并发请求合并：多个线程同时请求同一个 key 时，只允许第一个请求真正执行计算，其余请求等待并共享同一份执行结果。
2. 支持不同 key 并发隔离：不同 key 的请求应能并行执行，互不阻塞，不能因为某个 key 的慢请求影响其他 key。
3. 实现结果共享与清理：首个请求成功后，等待者获得相同结果；该 key 的进行中记录需要及时清理，后续新的同 key 请求应重新触发计算。
4. 实现失败不缓存：首个请求执行失败时，等待者应获得同一个失败结果，但失败不能被长期缓存，下一次同 key 请求应重新执行。
5. 支持超时等待与调用统计：等待者可以配置等待超时时间；合并器应能统计每个 key 的真实执行次数、共享命中次数和失败次数。
6. 为该模块编写完整的单元测试，覆盖正常流程（同 key 多线程只执行一次、不同 key 并行）、边界条件（单请求无合并、等待者超时、执行完成后再次请求）和异常分支（首请求失败后等待者收到错误、失败后再次请求重新执行）。测试代码放在 tests/singleflight/ 下。
7. 在该模块包内编写 README.md，说明模块功能、核心类的职责、并发合并流程以及使用示例。
