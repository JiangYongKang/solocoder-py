在当前 Python 项目中实现 API 密钥管理器，所有相关代码放在 src/solocoder_py/apikey/ 包下，使用内存数据结构存储密钥与使用记录。
1. 实现密钥生成与吊销：为指定主体（如用户或服务账号）生成一个唯一的 API 密钥。密钥本身由随机字符串组成，需具备足够的长度和熵值防止被猜测。密钥仅在全量生成时返回一次，后续只能通过密钥前缀片段识别而无法再次获取完整的原始密钥明文。支持按密钥 ID 或按主体批量吊销密钥，吊销后的密钥立即失效，任何后续使用该密钥的请求都应被拒绝。
2. 实现作用域权限绑定：每个 API 密钥在生成时绑定一组权限作用域（如只读、写入、管理员权限等）。请求到达时根据使用的密钥查询其权限信息，判断该请求是否在密钥的允许范围内。权限模型应支持通配符匹配（如 `resource:*` 匹配 `resource:read` 和 `resource:write`），以及作用域之间的包含关系（如管理员作用域天然包含只读和写入权限）。
3. 实现使用量追踪：每次 API 密钥被用于请求时记录使用事件。追踪维度包括：最近使用时间、总使用次数、一段时间窗口内的使用频率（如过去 1 小时的使用次数）。支持按密钥 ID 或按主体查询使用量统计，支持按使用频率对密钥列表排序（如找出最活跃或最不活跃的密钥）。超过指定天数未使用的密钥可以标记为闲置状态。
4. 为该模块编写完整的单元测试，覆盖正常流程（密钥生成后能用前缀识别、密钥校验通过返回权限信息、吊销后使用被拒绝、使用量追踪记录每次请求并正确统计、按权限作用域匹配允许和拒绝的资源访问）、边界条件（零权限作用域的密钥、通配符权限匹配边界、密钥 ID 不存在的查询返回、使用量统计窗口刚好切换时计数归零、同时为同一主体生成多个密钥互不影响）和异常分支（使用已吊销密钥的拒绝、使用不存在的密钥的拒绝、权限不足时拒绝具体资源访问、密钥前缀碰撞的处理、使用量统计在高并发下的计数准确性、密钥内容仅首次返回后续不可获取）。测试代码放在 tests/apikey/ 下。
5. 在该模块包内编写 README.md，说明模块功能、核心类的职责、密钥生命周期（生成-使用-吊销）、权限作用域模型与匹配规则、使用量追踪机制以及使用示例。
