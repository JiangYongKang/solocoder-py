你实现的内存块分配器存在一些缺陷，需要修复：
1. read 方法在不传 size 参数时返回整个块的全部字节，包括从未写入过的空白区域。调用方无法区分"显式写入了零"和"从未写入的空白内存"。需要让调用方能明确获知已写入的数据范围，或在接口层面避免返回未初始化的内存内容。
2. find_best_fit 在多个空闲块大小相同时总是选择地址最小的块，行为与 first_fit 完全一致。当空闲链表恰好按地址升序排列时，best_fit 策略退化成了 first_fit，无法产生可观测的策略差异。best_fit 需要定义等大块时的明确次级选择策略，使两种分配策略在合理场景下产生不同结果。
3. 修复后需要为上述两个改动补充对应的单元测试，验证 read 的已写入范围行为正确、best_fit 与 first_fit 在等大空闲块场景下产生可区分的结果。测试代码放在 tests/allocator/ 下。
4. 更新 README.md，说明 read 的写入范围追踪机制和 best_fit 等大块的次级选择策略。
5. 重新运行 poetry run pytest tests/allocator/ -q，确保所有测试通过。
